# SkPathRef

> 源文件: `include/private/SkPathRef.h`

## 概述

SkPathRef 头文件定义了路径几何形状识别和表示的核心数据结构。它提供了一套类型和结构体,用于识别和描述特殊几何形状(矩形、椭圆、圆角矩形等),以及路径的类型标记系统。这些定义是 Skia 路径优化渲染的基础,允许渲染引擎对特定形状采用优化的绘制路径。

## 架构位置

SkPathRef 位于 Skia 核心图形系统的路径表示层。它为 SkPath 类提供底层的形状识别支持,是路径优化和高效渲染的关键组件。该头文件定义在 `include/private` 目录,主要供 Skia 内部使用,特别是 GPU 后端优化路径绘制时需要这些信息。

## 主要类与结构体

### SkPathRectInfo

存储矩形路径的识别信息。

**关键成员变量**:

| 变量名 | 类型 | 说明 |
|--------|------|------|
| fRect | SkRect | 矩形的边界坐标 |
| fDirection | SkPathDirection | 路径方向(顺时针或逆时针) |
| fStartIndex | uint8_t | 起始顶点索引(0-3) |

**用途**: 当 `SkPath::isRect()` 返回 true 时,此结构体提供矩形的详细信息。

### SkPathOvalInfo

存储椭圆路径的识别信息。

**关键成员变量**:

| 变量名 | 类型 | 说明 |
|--------|------|------|
| fBounds | SkRect | 椭圆的外接矩形 |
| fDirection | SkPathDirection | 路径方向(顺时针或逆时针) |
| fStartIndex | uint8_t | 起始点索引,对虚线化很重要 |

**用途**: 当 `SkPath::isOval()` 返回 true 时,此结构体提供椭圆的详细信息。

### SkPathRRectInfo

存储圆角矩形路径的识别信息。

**关键成员变量**:

| 变量名 | 类型 | 说明 |
|--------|------|------|
| fRRect | SkRRect | 圆角矩形对象(包含边界和圆角半径) |
| fDirection | SkPathDirection | 路径方向(顺时针或逆时针) |
| fStartIndex | uint8_t | 起始点索引 |

**用途**: 当 `SkPath::isRRect()` 返回 true 时,此结构体提供圆角矩形的详细信息。

### SkPathIsAType 枚举

定义路径可能的特殊类型标记。

```cpp
enum class SkPathIsAType : uint8_t {
    kGeneral,   // 通用路径,无特殊形状
    kOval,      // 椭圆形状
    kRRect,     // 圆角矩形形状
};
```

**类型说明**:

| 枚举值 | 含义 | 存储的额外信息 |
|--------|------|---------------|
| kGeneral | 通用路径 | 无额外数据 |
| kOval | 椭圆 | 方向和起始索引(通过边界推断椭圆) |
| kRRect | 圆角矩形 | 方向和起始索引(通过点和动词推断圆角半径) |

**特殊说明**:
- **矩形**: 没有特殊的类型标记,需要通过 `isRect()` 方法检查点和动词来推断
- **椭圆**: 路径边界即为椭圆边界,无需额外存储半径
- **圆角矩形**: 不存储圆角半径,在调用 `isRRect()` 时通过分析点和动词重新推断

### SkPathIsAData 结构体

存储特殊形状类型的额外数据。

**关键成员变量**:

| 变量名 | 类型 | 说明 |
|--------|------|------|
| fStartIndex | uint8_t | 起始顶点/点索引 |
| fDirection | SkPathDirection | 路径方向 |

**用途**: 与 SkPathIsAType 配合使用,存储椭圆和圆角矩形的方向和起始点信息。

## 设计理念

### 路径类型识别系统

Skia 的路径可以标记为特殊类型,主要目的是:

1. **性能优化**: GPU 后端可以对已知形状使用专门的着色器或绘制路径
2. **保留语义**: 即使路径由低级的直线/曲线表示,仍保留高级形状信息
3. **兼容性**: 所有路径都有完整的点和动词表示,即使不识别类型标记也能正确绘制

### 存储策略

不同形状的存储策略体现了空间和计算的权衡:

- **椭圆**: 只存储方向和起始索引,边界即为椭圆边界,节省空间
- **圆角矩形**: 不存储圆角半径,通过分析点数据重新计算,因为:
  - 圆角矩形的半径可以从点的位置推断
  - 避免存储 4 个圆角的 8 个半径值
  - 大多数情况下不需要查询圆角信息

### 起始索引的重要性

`fStartIndex` 字段对于虚线绘制(dashing)至关重要:
- 虚线图案需要知道从哪个点开始绘制
- 不同的起始点会产生不同的虚线图案
- 保留起始索引确保虚线效果的正确性和可预测性

## 内部实现细节

### 形状识别流程

以椭圆为例,路径被识别为椭圆需要满足:

1. **点数检查**: 必须有正确数量的点(通常是 13 个点,包含 4 个二次贝塞尔曲线)
2. **动词序列**: 动词序列必须匹配椭圆的标准模式
3. **几何验证**: 点的位置必须精确匹配椭圆方程
4. **闭合检查**: 路径必须是闭合的

如果所有条件满足,路径可以被标记为 kOval 类型。

### 类型标记的保留

Skia 尝试在路径操作中保留类型标记:
- **变换操作**: 对椭圆应用均匀缩放后仍是椭圆
- **合并操作**: 某些情况下合并两个矩形可能得到矩形
- **失效条件**: 添加额外点或应用非均匀变换会清除类型标记

### GPU 后端优化

GPU 后端(如 Ganesh 和 Graphite)利用这些类型信息:
- **椭圆着色器**: 使用解析的椭圆方程,比光栅化贝塞尔曲线更快
- **矩形快速路径**: 可以使用简单的三角形扇形绘制
- **圆角矩形**: 使用专门的片段着色器计算距离场

## 设计模式与设计决策

### 标记而非继承

不使用继承体系(如 PathRect, PathOval 等)的原因:
- **统一接口**: 所有路径使用相同的 SkPath 接口
- **类型转换**: 避免频繁的向下转型和类型检查
- **内存效率**: 类型标记只占用 1 字节,远小于虚函数表指针

### 延迟计算

某些信息(如圆角矩形的半径)采用延迟计算策略:
- **优点**: 节省内存,大多数情况下不需要这些信息
- **缺点**: 查询时需要重新计算,有一定的 CPU 开销
- **权衡**: Skia 选择了内存优先的策略

### 信息结构体分离

将识别信息(Info 结构体)与路径类型(IsAType)分离:
- **清晰的职责**: 类型标记用于快速判断,Info 结构体用于详细信息
- **灵活性**: 可以在不同场景使用不同级别的信息
- **向后兼容**: 添加新的 Info 字段不影响类型系统

## 性能考量

### 内存占用

形状类型标记非常紧凑:
- SkPathIsAType: 1 字节
- SkPathIsAData: 2 字节(1 字节索引 + 1 字节方向)
- 总计只增加约 3 字节的路径元数据

### 识别成本

形状识别的 CPU 成本:
- **一次性成本**: 只在路径创建时或修改后执行
- **快速路径**: 对于通过构造函数创建的特殊形状,直接设置标记,无需验证
- **验证成本**: 对于通过点和动词构建的路径,需要完整的几何验证

### 渲染加速

类型信息带来的性能提升:
- **GPU 椭圆**: 比光栅化贝塞尔曲线快 2-5 倍
- **GPU 圆角矩形**: 使用距离场比传统方法快约 3 倍
- **CPU 裁剪**: 已知形状的裁剪可以使用解析方法
