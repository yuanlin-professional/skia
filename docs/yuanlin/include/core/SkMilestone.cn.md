# SkMilestone

> 源文件: `include/core/SkMilestone.h`

## 概述
SkMilestone 是 Skia 版本标识系统的核心组件，通过简单的宏定义机制记录和管理 Skia 库的里程碑版本号。该头文件提供全局版本常量，用于跟踪 Skia 的发布周期和版本演进。

## 架构位置
位于 Skia 核心模块 (`include/core`) 的最上层，作为全局配置信息的一部分，不依赖其他 Skia 组件，被整个库的构建系统和版本管理工具引用。

## 宏定义

### `SK_MILESTONE`
```cpp
#define SK_MILESTONE 148
```

**功能**: 定义当前 Skia 库的里程碑版本号。

**用途**:
- 版本标识和跟踪
- 构建系统中的版本检查
- API 兼容性验证
- 发布管理和文档生成

## 设计决策

### 简单宏定义设计
使用最简单的预处理器宏定义来存储版本号，而不是使用复杂的类或常量表达式。这种设计具有以下优势：
- **编译时可见**: 版本号在预处理阶段即可确定
- **跨语言兼容**: 可被 C/C++ 及构建脚本直接访问
- **零运行时开销**: 无需额外的内存或初始化逻辑
- **易于自动化**: 构建脚本可轻松修改和更新

### 递增版本号策略
Skia 采用递增的里程碑编号系统（当前为 148），这表明：
- 每个重要版本发布对应一个里程碑号
- 版本号的递增是线性的，便于追踪发展历史
- 不采用语义化版本号（如 x.y.z），而是使用单一数字标识

## 使用场景

### 版本检查
```cpp
// 在代码中检查最低版本要求
#if SK_MILESTONE >= 140
    // 使用新版本特性
#else
    // 使用旧版本兼容代码
#endif
```

### 条件编译
开发者可基于里程碑号进行条件编译，确保代码在不同版本间的兼容性：
```cpp
#ifdef SK_MILESTONE
    #if SK_MILESTONE < 150
        // 旧版本特定实现
    #endif
#endif
```

### 构建系统集成
构建脚本可读取此宏来：
- 生成版本字符串
- 创建发布标签
- 执行版本相关的测试
- 生成变更日志
