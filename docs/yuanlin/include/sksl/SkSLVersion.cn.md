# SkSL 版本定义

> 源文件: `include/sksl/SkSLVersion.h`

## 概述

`SkSLVersion.h` 定义了 SkSL (Skia Shading Language) 编译器支持的目标着色器语言版本。该文件通过枚举类型指定了不同的 GLSL 和 WebGL 版本,用于控制着色器编译器生成与特定平台兼容的着色器代码。这是 SkSL 跨平台支持的基础组件。

## 架构位置

本模块位于 Skia 的 SkSL 子系统核心层,属于编译器配置和目标平台定义部分。它为 SkSL 编译器提供了版本选择机制,使得同一份 SkSL 源代码可以编译为不同版本的 GLSL,从而在不同的 OpenGL 环境中运行。

## 主要类与结构体

### Version (枚举类)

**职责**: 定义 SkSL 编译器支持的目标着色器语言版本。

**枚举值说明**:

| 枚举值 | 说明 | 对应标准 |
|--------|------|---------|
| `k100` | 桌面端 GLSL 1.10、GLSL ES 1.00、WebGL 1.0 | OpenGL 2.0 / OpenGL ES 2.0 时代的着色器标准 |
| `k300` | 桌面端 GLSL 3.30、GLSL ES 3.00、WebGL 2.0 | OpenGL 3.3 / OpenGL ES 3.0 时代的现代着色器标准 |

### 版本特性对比

**GLSL 1.00 / 1.10 (k100) 特性**:
- 基础的顶点和片段着色器支持
- 有限的整数运算支持
- `attribute` 和 `varying` 关键字
- 较少的内置函数和特性
- 广泛的硬件兼容性(适用于旧设备)

**GLSL 3.00 / 3.30 (k300) 特性**:
- 增强的整数和位运算支持
- 统一使用 `in` 和 `out` 关键字
- 多渲染目标 (MRT) 支持
- 更丰富的内置函数库
- 纹理采样函数的改进
- 整数纹理和无符号整数支持

## 公共 API 使用

### 枚举值使用示例

```cpp
// 在编译器配置中指定目标版本
SkSL::Version targetVersion = SkSL::Version::k300;

// 根据版本选择不同的编译策略
if (targetVersion == SkSL::Version::k100) {
    // 使用旧版 GLSL 语法和特性
    // 生成兼容性更好但功能受限的代码
} else if (targetVersion == SkSL::Version::k300) {
    // 使用现代 GLSL 语法和特性
    // 生成功能更强大的代码
}
```

### 版本选择策略

编译器根据目标平台自动选择合适的版本:

1. **WebGL 1.0 环境**: 自动使用 `k100`
2. **WebGL 2.0 环境**: 自动使用 `k300`
3. **移动设备**: 根据 OpenGL ES 版本选择
4. **桌面平台**: 通常使用 `k300` 以获得最佳性能

## 内部实现细节

### 编译器版本适配

SkSL 编译器根据指定的 `Version` 枚举值执行以下操作:

1. **语法转换**: 将 SkSL 语法转换为目标版本的 GLSL 语法
   - `k100`: 使用 `attribute`/`varying` 关键字
   - `k300`: 使用 `in`/`out` 关键字

2. **特性检查**: 验证使用的语言特性是否在目标版本中可用
   - 整数运算: 仅在 `k300` 中完全支持
   - 高级纹理函数: 某些仅在 `k300` 中可用

3. **内置函数映射**: 将 SkSL 内置函数映射到目标版本的等效函数
   - `texture()` 函数在不同版本中有不同的签名

4. **版本声明生成**: 在生成的 GLSL 代码开头插入正确的版本声明
   - `k100`: `#version 100` (ES) 或 `#version 110` (Desktop)
   - `k300`: `#version 300 es` (ES) 或 `#version 330` (Desktop)

### 跨版本兼容性处理

编译器实现了一系列策略来处理不同版本间的差异:

- **降级转换**: 当使用 `k100` 时,将某些高级特性转换为等效的低级实现
- **功能检测**: 编译时检测并拒绝在目标版本中不支持的特性
- **自动适配**: 对于某些跨版本差异,编译器自动生成适配代码

## 设计模式与设计决策

### 枚举类型的选择

使用 `enum class` 而不是传统 `enum` 的优势:

1. **强类型安全**: 防止隐式转换为整数,避免误用
2. **命名空间隔离**: 枚举值限定在 `Version::` 作用域内
3. **前向兼容**: 便于未来添加新版本而不污染全局命名空间

### 版本粒度

当前设计只支持两个主要版本,这是基于以下考虑:

1. **覆盖主流平台**: `k100` 和 `k300` 覆盖了绝大多数目标平台
2. **简化维护**: 过多的版本支持会增加编译器复杂度
3. **实用主义**: OpenGL 2.x/ES 2.0 和 OpenGL 3.3/ES 3.0 是最常用的两个版本分界点

### 未来扩展性

虽然当前只有两个版本,但设计预留了扩展空间:

```cpp
enum class Version {
    k100,
    k300,
    // 未来可能添加:
    // k400,  // GLSL 4.00 / GLSL ES 3.1
    // k450,  // GLSL 4.50 / Vulkan SPIR-V
};
```

## 平台相关说明

### WebGL 环境

- **WebGL 1.0**: 必须使用 `k100`,对应 GLSL ES 1.00
- **WebGL 2.0**: 必须使用 `k300`,对应 GLSL ES 3.00
- **浏览器差异**: 某些浏览器对版本声明的解析可能有细微差异

### 移动平台

- **Android**: 根据设备 OpenGL ES 版本选择
  - ES 2.0 设备: 使用 `k100`
  - ES 3.0+ 设备: 使用 `k300`
- **iOS**: 现代 iOS 设备通常支持 `k300`

### 桌面平台

- **Windows/Linux**: 根据驱动程序支持选择
- **macOS**: OpenGL 已弃用,通常通过 Metal 后端渲染

## 使用建议

### 版本选择指南

1. **最大兼容性**: 使用 `k100` 以支持最广泛的设备
2. **最佳性能**: 在支持的情况下使用 `k300`
3. **功能需求**: 如果需要整数运算或高级特性,必须使用 `k300`
4. **目标平台**: 根据目标平台的 OpenGL 版本选择

### 常见陷阱

1. **隐式假设**: 不要假设所有平台都支持 `k300`
2. **特性检测**: 在使用高级特性前检查目标版本
3. **测试覆盖**: 在不同版本下测试着色器以确保兼容性

## 总结

`SkSLVersion.h` 虽然代码量很少,但在 SkSL 编译器架构中扮演着关键角色。它提供了一个清晰、类型安全的机制来指定目标着色器语言版本,使得 Skia 能够在不同的 OpenGL 环境中生成兼容的着色器代码。其简洁的设计体现了"少即是多"的原则,同时保留了未来扩展的灵活性。
