# Int96 -- 96 位有符号整数运算

> 源文件: `modules/bentleyottmann/src/Int96.cpp`

## 概述

`Int96.cpp` 实现了一个 96 位有符号整数类型 `Int96`，用于 Bentley-Ottmann 线段交点算法中需要超高精度整数运算的场景。该类型由一个 64 位有符号高位部分 (`hi`) 和一个 32 位无符号低位部分 (`lo`) 组成，总共提供 96 位的整数精度。这在计算两条线段交点坐标时，需要对 64 位和 32 位整数进行乘法运算而不丢失精度的情况下至关重要。

## 架构位置

`Int96` 位于 `modules/bentleyottmann/` 模块中，属于 Skia 的计算几何基础设施。Bentley-Ottmann 算法用于高效地检测一组线段的所有交点，而精确的整数运算是确保算法数值稳定性的关键。`Int96` 作为该模块的底层数学工具，为上层的交点计算提供精确的 96 位乘法和加法运算。

## 主要类与结构体

### `Int96`
```cpp
struct Int96 {
    int64_t hi;   // 高 64 位（有符号）
    uint32_t lo;  // 低 32 位（无符号）
};
```

该结构体表示一个 96 位有符号整数，采用高低位分离存储。`hi` 是有符号的，负数时为全 1（-1），正数时为 0 或正值。`lo` 始终作为无符号值处理。

## 公共 API 函数

### `Int96::Make(int32_t a)`
将 32 位有符号整数扩展为 96 位。正数时 `hi` 为 0，负数时 `hi` 为 -1（符号扩展）。

### `Int96::Make(int64_t a)`
将 64 位有符号整数扩展为 96 位。高 32 位通过右移提取，低 32 位通过掩码截取。

### `operator==` / `operator<`
使用 `std::tie` 进行字典序比较，先比较 `hi` 再比较 `lo`，实现 96 位整数的相等和小于判断。

### `operator+`
96 位加法，手动处理低 32 位的进位传播到高 64 位。

### `multiply(int64_t a, int32_t b)` / `multiply(int32_t a, int64_t b)`
核心函数：将一个 64 位整数与一个 32 位整数相乘，产生 96 位结果。第二个重载通过交换参数委托给第一个。

## 内部实现细节

### 乘法算法

`multiply(int64_t a, int32_t b)` 的实现分为两步乘法：

1. **低位无符号乘法**：将 `a` 的低 32 位与 `b`（转为无符号）相乘，得到 64 位中间结果 `newLo`。使用无符号乘法是为了避免有符号溢出的未定义行为。

2. **高位有符号乘法**：将 `a` 的高 32 位与 `b` 相乘，得到 64 位的高位结果 `newHi`。

3. **进位合并**：将 `newLo` 的高 32 位（溢出部分）加到 `newHi` 中。

4. **负数补偿**：如果 `b` 为负数，由于步骤 1 将 `b` 当作无符号数处理，需要从 `newHi` 中减去 `a` 的低 32 位进行补偿。这是因为无符号值 `(uint32_t)b` 等于 `b + 2^32`，多出的 `2^32 * loA` 需要扣除。

### 比较运算

比较运算利用 `std::tie` 创建元组进行字典序比较，这是正确的，因为 `hi` 是有符号的（决定整体正负），而 `lo` 是无符号的（在 `hi` 相同时决定大小）。

## 设计模式与设计决策

1. **值类型设计**：`Int96` 是一个简单的 POD 结构体，没有构造函数/析构函数开销，适合高频创建和按值传递。

2. **工厂方法而非构造函数**：使用 `Make` 静态方法替代构造函数，明确表达从较小类型到 96 位的扩展语义。

3. **混合符号存储**：高位使用有符号 `int64_t`，低位使用无符号 `uint32_t`，这样可以直接利用字典序比较的正确性，同时简化了溢出处理。

4. **非成员运算符**：比较和算术运算符定义为自由函数而非成员函数，符合对称二元运算的 C++ 惯用法。

## 性能考量

- 所有运算都是纯整数运算，没有浮点数转换或堆分配，执行速度非常快。
- `multiply` 函数仅使用 2 次乘法和少量加减法，避免了更通用但更慢的大整数库。
- 比较运算使用 `std::tie`，编译器通常能将其优化为直接的整数比较序列。
- 96 位的精度经过精确计算，刚好满足 Bentley-Ottmann 算法中 64 位 x 32 位乘法的需求，不会浪费存储空间。
