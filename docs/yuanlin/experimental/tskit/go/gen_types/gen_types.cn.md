# gen_types 类型生成器

> 源文件: experimental/tskit/go/gen_types/gen_types.go

## 概述

`gen_types` 是 TSKit 框架中的代码生成工具,负责从 C++ Emscripten 绑定代码中自动提取类型信息并生成对应的 TypeScript ambient namespace 声明文件。该工具通过解析特殊的宏标记和 Emscripten 绑定语法,自动化了类型定义的生成过程,大大减少了手动维护类型定义的工作量,并确保了 C++ 和 TypeScript 类型的一致性。

工具的核心功能包括:
1. 解析 C++ 文件中的 `TS_PRIVATE_EXPORT` 和 `TS_EXPORT` 宏,提取函数签名
2. 识别 Emscripten 的 `class_<>` 定义,生成类接口和构造函数接口
3. 解析 `value_object` 定义,生成结构体接口
4. 提取常量定义,生成只读属性声明
5. 按字母顺序组织输出,确保生成结果的确定性和可重复性

这个工具体现了元编程和自动化工具设计的最佳实践,特别是在跨语言类型系统同步方面的应用。

## 主要类与结构体

### type wasmObject

表示通过 Emscripten `class_<>` 绑定的 C++ 类。

```go
type wasmObject struct {
    name           string
    constructors   []string
    publicMethods  []string
    privateMethods []string
}
```

**字段说明:**
1. **name**: 类名,如 `"Something"`, `"AnotherClass"`
2. **constructors**: 构造函数签名列表,如 `["new(): Something", "new(name: string): Something"]`
3. **publicMethods**: 公共方法签名列表,如 `["getName(): string"]`
4. **privateMethods**: 私有方法签名列表,如 `["_setName(name: string): void"]`

**生成的 TypeScript 结构:**
```typescript
// 构造函数接口
export interface SomethingConstructor {
    new(): Something;
    new(name: string): Something;
}

// 实例接口
export interface Something extends embind.EmbindObject<Something> {
    _setName(name: string): void;  // 私有方法
    getName(): string;              // 公共方法
}
```

### type valueObject

表示通过 Emscripten `value_object<>` 绑定的 C++ 结构体。

```go
type valueObject struct {
    name   string
    fields []string
}
```

**字段说明:**
1. **name**: 结构体名,如 `"SomeValueObject"`
2. **fields**: 字段定义列表,如 `["alpha: number", "beta?: string"]`

**生成的 TypeScript 结构:**
```typescript
export interface SomeValueObject {
    alpha: number,
    beta?: string,
}
```

## 公共 API 函数

### main()

程序入口,解析命令行参数并处理所有输入文件。

**命令行参数:**
- `--input_cpp_dir`: 包含 C++ 绑定文件的目录路径
- `--output_namespace_dir`: 输出 ambient namespace 文件的目录路径

**执行流程:**
1. 解析命令行参数
2. 读取输入目录中的所有 `.cpp` 文件
3. 对每个文件调用 `generateAmbientNamespace`
4. 将生成的内容写入对应的 `.d.ts` 文件

**使用示例:**
```bash
go run gen_types.go \
  --input_cpp_dir=../../bindings \
  --output_namespace_dir=../../bindings
```

### generateAmbientNamespace(namespace, contents string) (string, error)

核心函数,解析 C++ 绑定代码并生成 TypeScript ambient namespace。

**参数:**
- `namespace`: 命名空间名称,通常是文件名去掉 `.cpp` 扩展名
- `contents`: C++ 文件的完整内容

**返回值:**
- `string`: 生成的 TypeScript 代码
- `error`: 解析错误或验证错误

**解析流程:**
1. 逐行扫描输入内容
2. 使用正则表达式匹配特殊标记和绑定语法
3. 提取函数、类、值对象和常量定义
4. 按字母顺序排序所有元素
5. 生成格式化的 TypeScript 代码

**输出结构:**
```typescript
/// <reference path="embind.d.ts" />
/* This file is autogenerated using gen_types.go and make generate */
declare namespace namespace_name {
    export interface Bindings {
        // 私有模块函数
        // 公共模块函数
        // 类构造函数引用
        // 常量
    }

    // 构造函数接口
    // 类实例接口
    // 值对象接口
}
```

### ensureSemicolon(js string) string

工具函数,确保 JavaScript/TypeScript 语句以分号结尾。

**参数:**
- `js`: JavaScript/TypeScript 语句

**返回值:**
- `string`: 以分号结尾的语句

**实现:**
```go
func ensureSemicolon(js string) string {
    if !strings.HasSuffix(js, ";") {
        return js + ";"
    }
    return js
}
```

## 内部实现细节

### 正则表达式解析器

工具定义了多个正则表达式来匹配不同的绑定模式:

#### 1. privateExportLine

```go
privateExportLine = regexp.MustCompile(`TS_PRIVATE_EXPORT\("(?P<export>.+)"\)`)
```

**匹配示例:**
```cpp
TS_PRIVATE_EXPORT("_privateFunction(x: number, y: number): number")
```

**提取结果:**
- `export` = `"_privateFunction(x: number, y: number): number"`

#### 2. publicExportLine

```go
publicExportLine = regexp.MustCompile(`TS_EXPORT\("(?P<export>.+)"\)`)
```

**匹配示例:**
```cpp
TS_EXPORT("publicFunction(input: string): void")
```

**提取结果:**
- `export` = `"publicFunction(input: string): void"`

#### 3. classDefinitionStart

```go
classDefinitionStart = regexp.MustCompile(`class_<.+>\("(?P<name>.+)"\)`)
```

**匹配示例:**
```cpp
class_<Something>("Something")
```

**提取结果:**
- `name` = `"Something"`

#### 4. classDefinitionEnd

```go
classDefinitionEnd = regexp.MustCompile(`\.(function|constructor).+;`)
```

**匹配示例:**
```cpp
.function("getName", &Something::getName);
```

**作用:**
- 标记类定义的结束
- 重置 `currentWasmObject` 状态

#### 5. typeAnnotation

```go
typeAnnotation = regexp.MustCompile(`@type\s+(?P<optional>@optional)?\s*(?P<type>\w+)`)
```

**匹配示例:**
```cpp
/** @type number */
/** @type @optional string */
```

**提取结果:**
- `optional` = `"@optional"` 或 `""`
- `type` = `"number"` 或 `"string"`

#### 6. valueObjectStart

```go
valueObjectStart = regexp.MustCompile(`value_object<.+>\("(?P<name>.+)"\)`)
```

**匹配示例:**
```cpp
value_object<SomeValueObject>("SomeValueObject")
```

**提取结果:**
- `name` = `"SomeValueObject"`

#### 7. valueObjectField

```go
valueObjectField = regexp.MustCompile(`.field\("(?P<name>.+?)",.+\);?`)
```

**匹配示例:**
```cpp
.field("columns", &SomeValueObject::columns)
.field("name", &SomeValueObject::slot);
```

**提取结果:**
- `name` = `"columns"` 或 `"name"`

#### 8. constantDefinition

```go
constantDefinition = regexp.MustCompile(`constant\("(?P<name>.+?)",.+\);`)
```

**匹配示例:**
```cpp
constant("SOME_FLAG", 0x2);
```

**提取结果:**
- `name` = `"SOME_FLAG"`

### 解析状态机

函数维护多个状态变量来跟踪解析进度:

```go
var currentWasmObject *wasmObject      // 当前正在解析的类
var currentValueObject *valueObject    // 当前正在解析的值对象
var lastTypeAnnotation string          // 最近遇到的类型注解
var lastTypeOptional bool              // 最近的类型是否可选
```

**状态转换:**
1. 遇到 `class_<>` → 创建新的 `wasmObject`
2. 遇到 `.function` 或 `.constructor` 结尾 → 结束类定义
3. 遇到 `value_object<>` → 创建新的 `valueObject`
4. 遇到 `.field` 结尾的分号 → 结束值对象定义

### 输出生成逻辑

#### 1. Bindings 接口生成

```go
output := fmt.Sprintf(`/// <reference path="embind.d.ts" />
/* This file is autogenerated using gen_types.go and make generate */
declare namespace %s {
    export interface Bindings {
`, namespace)
```

**包含内容:**
- 私有模块函数(按字母顺序)
- 公共模块函数(按字母顺序)
- 类构造函数引用(只读)
- 常量定义(只读)

#### 2. 构造函数接口生成

```go
for _, obj := range wasmObjects {
    sort.Strings(obj.constructors)
    output += fmt.Sprintf("\texport interface %sConstructor {\n", obj.name)
    for _, c := range obj.constructors {
        output += fmt.Sprintf("\t\t%s\n", ensureSemicolon(c))
    }
    output += "\t}\n\n"
}
```

**生成格式:**
```typescript
export interface SomethingConstructor {
    new(name: string): Something;
}
```

#### 3. 类实例接口生成

```go
for _, obj := range wasmObjects {
    sort.Strings(obj.privateMethods)
    sort.Strings(obj.publicMethods)
    output += fmt.Sprintf("\texport interface %s extends embind.EmbindObject<%s> {\n", obj.name, obj.name)
    // 私有方法
    for _, m := range obj.privateMethods {
        output += fmt.Sprintf("\t\t%s\n", ensureSemicolon(m))
    }
    if len(obj.privateMethods) > 0 {
        output += "\n"
    }
    // 公共方法
    for _, m := range obj.publicMethods {
        output += fmt.Sprintf("\t\t%s\n", ensureSemicolon(m))
    }
    output += "\t}\n\n"
}
```

**生成格式:**
```typescript
export interface Something extends embind.EmbindObject<Something> {
    _setName(name: string): void;

    getName(): string;
}
```

#### 4. 值对象接口生成

```go
for _, obj := range valueObjects {
    sort.Strings(obj.fields)
    output += fmt.Sprintf("\texport interface %s {\n", obj.name)
    for _, f := range obj.fields {
        output += fmt.Sprintf("\t\t%s,\n", f)
    }
    output += "\t}\n\n"
}
```

**生成格式:**
```typescript
export interface SomeValueObject {
    alpha: number,
    beta: string,
    gamma?: number,
}
```

### 错误处理

工具包含多个验证点:

#### 1. 字段缺少类型注解

```go
if match := valueObjectField.FindStringSubmatch(line); match != nil {
    if currentValueObject == nil {
        return "", skerr.Fmt("Line %d: Found a .field outside a value object declaration", i+1)
    }
    if lastTypeAnnotation == "" {
        return "", skerr.Fmt("Line %d: field %q must be preceded by a @type annotation", i+1, match[1])
    }
    // ...
}
```

**错误示例:**
```cpp
.field("misbehaving_field", &Obj::field)  // 缺少 @type 注解
```

#### 2. 常量缺少类型注解

```go
if match := constantDefinition.FindStringSubmatch(line); match != nil {
    if lastTypeAnnotation == "" {
        return "", skerr.Fmt("Line %d: constant %q must be preceded by a @type annotation", i+1, match[1])
    }
    // ...
}
```

**错误示例:**
```cpp
constant("BAD_CONSTANT", 42);  // 缺少 @type 注解
```

### 字母顺序排序

所有生成的元素都按字母顺序排序,确保输出稳定:

```go
sort.Strings(privateModuleFunctions)
sort.Strings(publicModuleFunctions)
sort.Slice(wasmObjects, func(i, j int) bool {
    return wasmObjects[i].name < wasmObjects[j].name
})
sort.Slice(valueObjects, func(i, j int) bool {
    return valueObjects[i].name < valueObjects[j].name
})
sort.Strings(constants)
```

**好处:**
1. 便于 diff 和版本控制
2. 易于查找特定的定义
3. 减少合并冲突

## 设计模式与设计决策

### 1. 解释器模式 (Interpreter Pattern)

工具实现了一个简化的解释器,解析特定的 DSL:
- 输入: C++ + 特殊宏
- 输出: TypeScript 类型定义

### 2. 构建器模式 (Builder Pattern)

逐步构建输出字符串:
```go
output := "header"
output += "bindings interface"
output += "constructor interfaces"
output += "class interfaces"
output += "value object interfaces"
```

### 3. 状态模式 (State Pattern)

使用状态变量跟踪解析上下文:
- `currentWasmObject`: 当前类
- `currentValueObject`: 当前值对象
- `lastTypeAnnotation`: 类型注解缓存

### 4. 模板方法模式

`generateAmbientNamespace` 定义了固定的生成流程:
1. 解析输入
2. 排序元素
3. 生成输出

### 5. 单一职责原则

每个函数职责明确:
- `main`: 处理 I/O 和命令行参数
- `generateAmbientNamespace`: 解析和生成
- `ensureSemicolon`: 格式化辅助

### 6. 约定优于配置

- 输入文件必须是 `.cpp`
- 输出文件名 = 输入文件名 + `.d.ts`
- 命名空间名 = 文件名(无扩展名)

## 性能考量

### 1. 文件 I/O

```go
contents, err := ioutil.ReadFile(fp)
```
- 一次性读取整个文件到内存
- 适合小到中等大小的文件(<10MB)
- 大文件可能需要流式处理

### 2. 正则表达式编译

```go
var (
    privateExportLine    = regexp.MustCompile(...)
    publicExportLine     = regexp.MustCompile(...)
    // ...
)
```
- 正则表达式在包初始化时编译一次
- 编译结果被所有调用复用
- 避免重复编译开销

### 3. 字符串拼接

```go
output += fmt.Sprintf("\t\t%s\n", ...)
```
- 使用 `+=` 运算符拼接字符串
- 对于少量拼接(<1000次)可接受
- 大量拼接应使用 `strings.Builder`

**优化建议:**
```go
var builder strings.Builder
builder.WriteString("...")
return builder.String()
```

### 4. 排序开销

```go
sort.Strings(privateModuleFunctions)
sort.Slice(wasmObjects, ...)
```
- 时间复杂度: O(n log n)
- 对于典型的绑定文件(<100个元素),开销可忽略
- 换取确定性输出的代价合理

### 5. 逐行扫描

```go
lines := strings.Split(contents, "\n")
for i, line := range lines { ... }
```
- 一次性分割所有行
- 内存开销: O(n),n 为文件大小
- 适合大多数情况
