# TSKit ESLint 配置

> 源文件: `experimental/tskit/.eslintrc.js`

## 概述

`.eslintrc.js` 是 Skia TSKit 项目的 ESLint 配置文件，定义了 TypeScript 代码的静态检查规则。它基于 `airbnb-typescript` 规则集，并启用了需要类型信息的高级检查规则。

## 架构位置

位于 `experimental/tskit/` 目录，作为 TSKit（TypeScript Kit）实验性项目的代码质量配置。TSKit 是 Skia 的 WebAssembly/TypeScript 绑定工具链的原型。

## 主要类与结构体

无。该文件导出一个 ESLint 配置对象。

## 公共 API 函数

无。通过 `module.exports` 导出配置对象。

## 内部实现细节

1. **解析器**: 使用 `@typescript-eslint/parser`，配合 `tsconfig.json` 启用类型感知检查
2. **基础规则集**:
   - `airbnb-typescript`: Airbnb 的 TypeScript 规范
   - `@typescript-eslint/recommended-requiring-type-checking`: 需要类型信息的推荐规则
3. **关闭的规则**:
   - `no-explicit-any`: 允许使用 `any` 类型（WebAssembly 绑定需要）
   - `no-namespace`: 允许 TypeScript 命名空间
   - `no-unsafe-call/member-access`: 允许对 `any` 类型的调用和成员访问
   - `triple-slash-reference`: 允许三斜杠引用指令
   - `indent`: 关闭缩进检查
   - `import/prefer-default-export`: 允许具名导出
   - `no-param-reassign` / `no-underscore-dangle`: 放宽参数和命名限制
4. **React 设置**: 设置 `version: "latest"` 以抑制 React 版本警告（项目不使用 React）

## 依赖关系

- `@typescript-eslint/parser` 和 `@typescript-eslint/eslint-plugin`
- `eslint-config-airbnb-typescript`
- 本地 `tsconfig.json` 文件

## 设计模式与设计决策

- 使用 `root: true` 防止 ESLint 向上层目录查找配置
- 关闭多个严格规则以适应 WebAssembly 绑定代码的特殊需求（大量 `any` 类型和 unsafe 操作）

## 性能考量

- 启用了类型感知规则（`recommended-requiring-type-checking`），需要额外的 TypeScript 编译时间

## 相关文件

- `experimental/tskit/bindings/`: TypeScript 绑定源文件
- `experimental/tskit/tsconfig.json`: TypeScript 编译配置
