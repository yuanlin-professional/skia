# SkDCubicToQuads

> 源文件: src/pathops/SkDCubicToQuads.cpp

## 概述

`SkDCubicToQuads.cpp` 是 Skia 路径操作模块中的一个实用工具文件,提供了将三次贝塞尔曲线(Cubic Bezier Curve)近似转换为二次贝塞尔曲线(Quadratic Bezier Curve)的功能。该文件实现了基于度升(degree elevation)的逆向算法,通过数学推导从三次曲线的控制点计算出最佳拟合的二次曲线控制点。

这种转换在某些图形API只支持二次曲线时非常有用,或者用于简化曲线以提高性能。虽然三次曲线无法被单个二次曲线完全精确表示,但该算法通过平均两个可能的控制点位置来获得最佳近似。该功能主要用于测试和特殊场景。

## 架构位置

`SkDCubicToQuads.cpp` 位于路径操作的曲线工具层:

- **模块路径**: `src/pathops/`
- **功能层级**: 曲线转换工具
- **用途**: 测试和曲线简化(标注为 "used for testing only")
- **依赖组件**:
  - `SkPathOpsCubic.h`: 三次曲线表示
  - `SkPathOpsQuad.h`: 二次曲线表示
  - `SkPathOpsPoint.h`: 点类型
- **被使用者**:
  - 测试代码
  - 特殊的曲线简化场景

该文件提供了曲线类型之间转换的基础设施,虽然代码量极少,但数学原理严谨。

```

```

### SkDQuad (二次曲线类)

二次贝塞尔曲线的双精度表示,包含3个控制点。

**成员**:
```cpp
SkDPoint[3];  // P0, P1, P2
```

## 公共 API 函数

### `SkDQuad SkDCubic::toQuad() const`

将三次贝塞尔曲线转换为最佳拟合的二次贝塞尔曲线。

**返回值**: 近似的二次曲线 `SkDQuad`

**用途**: 测试和需要降低曲线次数的场景

**注意**: 这是近似转换,可能存在误差,不适合精确的几何计算。

## 内部实现细节

### 1. 数学原理

#### 度升公式(Degree Elevation)

从二次曲线升级到三次曲线的标准公式:

给定二次曲线控制点 P0, P1, P2,升级后的三次曲线控制点为:
```
Q0 = P0
Q1 = (1/3)P0 + (2/3)P1
Q2 = (2/3)P1 + (1/3)P2
Q3 = P2
```

#### 度降算法(Degree Reduction)

给定三次曲线控制点 Q0, Q1, Q2, Q3,需要反求 P0, P1, P2。

从度升公式可推导出 P1 的两种计算方式:
```
方法1: P1 = (3/2)Q1 - (1/2)Q0
方法2: P1 = (3/2)Q2 - (1/2)Q3
```

**问题**: 如果三次曲线不是由二次曲线升级而来,两种方法会得到不同的 P1。

**解决方案**: 对两个结果取平均,得到最佳近似:
```
P1 = [方法1 + 方法2] / 2
   = [3/2·Q1 - 1/2·Q0 + 3/2·Q2 - 1/2·Q3] / 2
   = -1/4·Q0 + 3/4·Q1 + 3/4·Q2 - 1/4·Q3
```

端点保持不变:
```
P0 = Q0
P2 = Q3
```

### 2. 代码实现

```cpp
SkDQuad SkDCubic::toQuad() const {
    SkDQuad quad;

    // 端点直接复制
    quad[0] = fPts[0];  // P0 = Q0

    // 从 Q1 计算中间控制点的候选位置
    const SkDPoint fromC1 = {
        (3 * fPts[1].fX - fPts[0].fX) / 2,  // (3/2)Q1 - (1/2)Q0
        (3 * fPts[1].fY - fPts[0].fY) / 2
    };

    // 从 Q2 计算中间控制点的候选位置
    const SkDPoint fromC2 = {
        (3 * fPts[2].fX - fPts[3].fX) / 2,  // (3/2)Q2 - (1/2)Q3
        (3 * fPts[2].fY - fPts[3].fY) / 2
    };

    // 取平均
    quad[1].fX = (fromC1.fX + fromC2.fX) / 2;
    quad[1].fY = (fromC1.fY + fromC2.fY) / 2;

    // 端点直接复制
    quad[2] = fPts[3];  // P2 = Q3

    return quad;
}
```

### 3. 计算步骤详解

**步骤1**: 计算 `fromC1`
- 使用三次曲线的第二个控制点 Q1
- 应用公式: P1 = (3/2)Q1 - (1/2)Q0
- 这是基于曲线起始部分的估计

**步骤2**: 计算 `fromC2`
- 使用三次曲线的第三个控制点 Q2
- 应用公式: P1 = (3/2)Q2 - (1/2)Q3
- 这是基于曲线结束部分的估计

**步骤3**: 平均两个估计
- `quad[1] = (fromC1 + fromC2) / 2`
- 最小化全局误差
- 在曲线起点和终点附近的拟合质量相当

### 4. 误差分析

该方法的误差特性:

**最佳情况**: 如果输入的三次曲线确实是由二次曲线度升得到的,该算法可以精确恢复原始二次曲线。

**一般情况**: 对于真正的三次曲线,存在近似误差:
- 误差最大处通常在曲线中部
- 端点和端点切线方向保持不变
- 曲率可能无法完全匹配

**不适用情况**:
- 具有拐点的三次曲线
- S形曲线
- 复杂扭曲的三次曲线

这些情况下单个二次曲线无法良好近似,需要分段逼近。

### 5. 使用场景

由于标注为 "used for testing only",该功能主要用于:

**测试验证**:
- 验证曲线度升算法的正确性
- 测试二次曲线和三次曲线的转换对称性
- 单元测试中的辅助工具

**特殊优化**:
- 当检测到三次曲线接近二次曲线时,进行简化
- 降低曲线复杂度以提高渲染速度
- 针对只支持二次曲线的图形后端

**不推荐用于**:
- 生产环境的精确几何计算
- 路径布尔运算的核心算法
- 需要精确控制误差的场景

## 设计模式与设计决策

### 1. 成员函数而非自由函数

将转换功能作为 `SkDCubic` 的成员方法:
- 面向对象的接口设计
- 清晰的所有权语义(从 this 转换)
- 便于未来扩展其他转换方法

### 2. 值返回而非引用参数

直接返回 `SkDQuad` 对象:
- 现代 C++ 的移动语义优化
- 接口更清晰简洁
- 避免输出参数的模糊性

### 3. 数学平均策略

采用两个估计的平均值:
- 最小化全局误差
- 算法简单高效
- 无需迭代优化

### 4. 保持端点不变

端点直接复制而非计算:
- 保证曲线连续性
- 避免数值误差累积
- 符合几何直觉

### 5. 最小化实现

整个文件仅45行,代码极其简洁:
- 单一职责原则
- 无冗余代码
- 易于理解和维护

## 性能考量

### 1. 时间复杂度

- **计算复杂度**: O(1),固定次数的算术运算
- **具体操作**: 12次乘法,8次加减法,2次除法
- **无循环或迭代**

### 2. 空间复杂度

- **栈空间**: O(1),仅局部变量
- **返回值**: 3个点(6个 double),共48字节
- **无动态分配**

### 3. 数值稳定性

**优点**:
- 仅涉及加减乘除,无复杂函数
- 除法的除数为常量(2),无除零风险
- 使用双精度浮点,精度充足

**潜在问题**:
- 当控制点坐标很大时,乘以3可能溢出(极端情况)
- 减法可能导致精度损失(相近值相减)

**实际使用**: 由于标注为测试用途,数值稳定性要求不严格。

### 4. 编译器优化

现代编译器可以充分优化该函数:
- 内联整个函数
- 向量化算术运算(SIMD)
- 消除临时变量

### 5. 缓存友好性

- 顺序访问 `fPts` 数组
- 局部性良好
- 无随机内存访问
