# SkCubicClipper

> 源文件
> - src/core/SkCubicClipper.h
> - src/core/SkCubicClipper.cpp

## 概述

`SkCubicClipper` 是 Skia 图形库中用于裁剪三次贝塞尔曲线的工具类。该类专门处理在 Y 轴方向上已经单调的三次贝塞尔曲线,根据给定的矩形裁剪区域对曲线进行裁剪。主要用于渲染管线中的几何体裁剪优化,确保只绘制可见区域内的曲线部分。

该类通过数学方法计算贝塞尔曲线与裁剪边界的交点,使用二分法或牛顿迭代法求解,从而高效地截取曲线在裁剪区域内的部分。

## 架构位置

`SkCubicClipper` 位于 Skia 的核心几何处理层,在渲染管线的早期阶段发挥作用:

```
Skia Core Layer
  └─ Geometry Processing
      ├─ SkPath (路径表示)
      ├─ SkGeometry (几何计算工具)
      └─ SkCubicClipper (三次曲线裁剪) ← 当前模块
```

该类在路径光栅化之前对曲线进行预处理,与 `SkGeometry` 模块紧密协作,是路径渲染优化的重要组成部分。

**核心职责**:

## 公共 API 函数

### 构造与配置

```cpp
SkCubicClipper()
```
默认构造函数,初始化时将裁剪区域设置为空。

```cpp
void setClip(const SkIRect& clip)
```
设置裁剪区域,将整数矩形转换为浮点矩形存储。

**参数**:
- `clip`: 整数类型的裁剪矩形

### 裁剪操作

```cpp
[[nodiscard]] bool clipCubic(const SkPoint src[4], SkPoint dst[4])
```
对三次贝塞尔曲线进行裁剪。

**参数**:
- `src`: 输入的 4 个控制点,必须在 Y 方向上单调
- `dst`: 输出裁剪后的 4 个控制点

**返回值**:
- `true`: 曲线在裁剪区域内有可见部分
- `false`: 曲线完全在裁剪区域外

**工作流程**:
1. 确保曲线在 Y 方向单调递减(必要时反转点序)
2. 快速检测:曲线完全在上方或下方则直接返回 false
3. 如果曲线部分超出上边界,计算交点并截取下半部分
4. 如果曲线部分超出下边界,计算交点并截取上半部分
5. 如果之前反转过点序,恢复原始顺序

### 静态工具函数

```cpp
[[nodiscard]] static bool ChopMonoAtY(const SkPoint pts[4], SkScalar y, SkScalar* t)
```
计算单调三次曲线与水平线 y 的交点参数 t。

**参数**:
- `pts`: 单调三次曲线的 4 个控制点
- `y`: 水平线的 Y 坐标
- `t`: 输出参数,交点对应的曲线参数 (0 ≤ t ≤ 1)

**返回值**:
- `true`: 成功找到交点且 t 在 [0,1] 范围内
- `false`: 未找到有效交点

## 内部实现细节

### 求解算法

代码提供两种求解方法(通过宏 `NEWTON_RAPHSON` 控制):

**1. 牛顿迭代法** (默认未启用):
- 二次收敛,通常 ≤ 3 次迭代
- 使用 Halley 方法的变体: `t_{n+1} = t_n - 2f(t_n)f'(t_n) / (2[f'(t_n)]^2 - f(t_n)f''(t_n))`
- 容差设置为 1/16384

**2. 二分法** (当前使用):
- 线性收敛,通常需要 16 次迭代
- 检查端点是否跨越零点
- 逐步缩小区间直到容差 1/65536
- 使用 De Casteljau 算法评估曲线值

### 曲线细分

当找到交点参数 t 后,使用 `SkChopCubicAt()` 函数(来自 SkGeometry 模块)在 t 处将三次曲线分割成两段,每段由 7 个点表示(两条曲线共享中间点)。

### 浮点数安全

- 所有比较操作都考虑了 NaN 安全性
- 使用 `SkScalarInterp()` 进行线性插值,保证数值稳定性

## 设计模式与设计决策

### 设计模式

1. **工具类模式**: 提供静态和实例方法,封装裁剪算法细节
2. **状态保持**: 裁剪矩形作为成员变量,避免重复传参
3. **就地修改**: 输入输出使用数组,减少内存分配

### 设计决策

**为何要求 Y 单调性**:
- 简化求解:单调曲线与水平线最多一个交点
- 性能优化:避免复杂的多根求解
- 现实需求:扫描线算法通常需要单调曲线

**选择二分法而非牛顿法**:
- 鲁棒性更好,不会发散
- 实现简单,易于维护
- 性能差异在实际应用中可忽略(16 次迭代仍然很快)

**容差选择**:
- 二分法容差 1/65536:平衡精度和性能,对于屏幕坐标足够精确
- 牛顿法容差 1/16384:更宽松,因为收敛更快

**反转机制**:
- 通过反转控制点确保下降顺序,统一处理逻辑
- 最后恢复顺序保持接口一致性

## 性能考量

### 优化策略

1. **快速拒绝测试**: 首先检查曲线是否完全在裁剪区域外,避免不必要的计算
2. **就地操作**: 直接修改目标数组,无需额外内存分配
3. **浮点转换缓存**: 整数裁剪区域转换为浮点后缓存在 fClip 中
4. **算法选择**: 二分法虽然迭代次数多,但每次迭代成本低且稳定

### 性能特征

- **时间复杂度**: O(1),固定迭代次数(最多 16 次)
- **空间复杂度**: O(1),只需临时数组存储 7 个点
- **典型开销**: 完全可见或完全不可见的快速路径几乎无开销,仅裁剪边界情况需要完整计算

### 潜在瓶颈

对于复杂路径包含大量曲线时,裁剪操作的累积开销可能成为瓶颈,但通过早期裁剪可以大幅减少后续渲染开销,整体上仍然是性能优化。
