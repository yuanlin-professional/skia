# SkMasks

> 源文件: src/core/SkMasks.h, src/core/SkMasks.cpp

## 概述

`SkMasks` 是 Skia 图形库中用于处理位掩码颜色格式的工具类。该类主要用于从像素数据中提取和转换 RGBA 颜色分量，特别是处理非标准位深度的颜色格式。它支持将任意位宽（1-8 位）的颜色分量转换为标准的 8 位格式，这在解码各种图像格式（如 BMP、ICO 等）时非常有用。

`SkMasks` 封装了颜色掩码信息，包括每个颜色通道的掩码值、位移量和位宽。通过这些信息，它能够从压缩的像素数据中正确提取颜色分量，并将其转换为标准的 8 位颜色值。该类采用不可变设计，一旦创建后其内部状态不可修改，保证了线程安全性。

## 架构位置

`SkMasks` 位于 Skia 的核心层（`src/core`），是图像解码流程中的基础工具类。它主要服务于：

- **图像编解码器**：在解码 BMP、ICO 等格式时处理非标准颜色格式
- **像素格式转换**：处理各种自定义位掩码颜色格式到标准格式的转换
- **颜色空间处理**：作为底层工具支持颜色数据的提取和转换

该类不依赖于高层的图形渲染模块，而是作为纯粹的数据转换工具存在，使其可以在各种图像处理场景中被复用。

## 主要类与结构体

### SkMasks 类

**继承关系**：无继承关系，独立工具类

| `size` | `uint32_t` | 掩码的位宽（1-8） |

### InputMasks 结构体

用于创建 `SkMasks` 对象时指定输入掩码：

| 成员 | 类型 | 说明 |

| `blue` | `uint32_t` | 蓝色通道原始位掩码 |
| `alpha` | `uint32_t` | Alpha 通道原始位掩码 |

## 公共 API 函数

### 静态工厂方法

```cpp
static SkMasks* CreateMasks(InputMasks masks, int bytesPerPixel);
```
创建 `SkMasks` 对象的工厂方法。验证输入掩码的合法性（不重叠、符合字节数限制），处理并转换原始掩码为内部 `MaskInfo` 格式。如果掩码非法则返回 `nullptr`。

### 颜色分量提取方法

```cpp
uint8_t getRed(uint32_t pixel) const;
uint8_t getGreen(uint32_t pixel) const;
uint8_t getBlue(uint32_t pixel) const;
uint8_t getAlpha(uint32_t pixel) const;
```
从像素值中提取对应的颜色分量，并转换为 8 位标准格式（0-255）。这些方法应用掩码、位移和位宽转换，确保输出统一的 8 位值。

### 获取器方法

```cpp
uint32_t getAlphaMask() const;
```
返回 Alpha 通道的原始掩码值，主要用于其他解码模式需要访问 Alpha 掩码信息的场景。

## 内部实现细节

### 位掩码处理机制

`process_mask()` 函数是核心处理逻辑：

1. **计算位移量**：通过计数尾部零位确定右移位数
2. **计算掩码宽度**：通过计数连续的 1 位确定有效位宽
3. **验证连续性**：检测掩码是否连续，非连续时发出警告但继续处理
4. **截断处理**：对于超过 8 位的掩码，调整位移量并截断到 8 位

### 位深度转换表

实现使用预计算的查找表 `n_bit_to_8_bit_lookup_table[]`，包含 1-7 位到 8 位的转换映射：

- **1 位**：0 → 0, 1 → 255
- **2 位**：均匀分布到 0, 85, 170, 255
- **3-7 位**：按比例缩放到完整的 0-255 范围

这种设计避免了运行时的除法和乘法运算，提高了性能。8 位值直接返回，无需转换。

### 颜色分量提取流程

`get_comp()` 辅助函数执行三步转换：

1. 应用掩码提取位段：`pixel & mask`
2. 右移到最低位：`>> shift`
3. 通过 `convert_to_8()` 转换为 8 位标准值

### 掩码验证逻辑

`CreateMasks()` 执行严格的验证：

- **字节对齐检查**：根据 `bytesPerPixel` 裁剪掩码，确保不超出像素边界
- **重叠检测**：通过位与操作检查任意两个通道是否有重叠位
- **范围验证**：断言 `bytesPerPixel` 在 1-4 之间

## 设计模式与设计决策

### 不可变对象模式

所有成员变量声明为 `const`，确保对象创建后状态不变。这带来以下优势：

- **线程安全**：多线程可安全共享同一实例
- **缓存友好**：对象可被缓存和重用
- **简化推理**：无需担心状态变化

### 工厂方法模式

使用静态 `CreateMasks()` 而非公共构造函数，提供以下好处：

- **验证输入**：在创建前检查掩码合法性
- **返回空指针**：非法输入时返回 `nullptr` 而非抛出异常
- **封装复杂性**：隐藏掩码处理的复杂逻辑

### 查找表优化

使用预计算的查找表而非运行时计算，体现了空间换时间的策略：

- **性能优先**：颜色分量提取是热点路径，需要极致性能
- **表大小可控**：总共只需 254 字节（2+4+8+16+32+64+128）
- **精确映射**：避免浮点运算带来的精度问题

### constexpr 构造函数

使用 `constexpr` 构造函数允许编译期常量初始化，减少运行时开销。

## 性能考量

### 查找表的性能优势

对于 1-7 位的转换，使用查找表避免了以下开销：

- **除法运算**：直接映射避免 `value * 255 / max_value` 的除法
- **条件分支**：表索引是无分支操作
- **缓存局部性**：查找表在内存中连续存储

### 内联优化

`getAlphaMask()` 等简单方法定义在头文件中，允许编译器内联优化，消除函数调用开销。

### 位运算效率

使用位运算（`&`, `>>`, `<<`）而非算术运算，充分利用现代 CPU 的位操作单元，通常只需 1-2 个时钟周期。

### 掩码验证的一次性成本

验证逻辑集中在 `CreateMasks()` 中，对象创建后的使用路径完全无验证开销，符合"构造时验证，使用时信任"的原则。
