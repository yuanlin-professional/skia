# SkRTree

> 源文件: src/core/SkRTree.h, src/core/SkRTree.cpp

## 概述

`SkRTree` 是 Skia 中实现的 R 树（R-Tree）数据结构，用于空间索引和快速空间查询。它继承自 `SkBBoxHierarchy` 接口，提供了一种高效的方式来管理和查询二维空间中的边界矩形（bounding boxes）集合。R 树是一种平衡的 n 叉树，包含边界矩形的层次结构，主要用于优化绘制命令的空间查询性能。

该实现采用 STR（Sort-Tile-Recursive）算法进行批量加载（bulk-loading），通过自底向上的方式一次性构建整个树结构，而不是逐个插入节点。这种批量加载策略显著提高了树的构建效率和查询性能。

## 架构位置

`SkRTree` 位于 Skia 的核心渲染管道中，主要用于优化绘图操作的空间查询。它在绘制记录（picture recording）和回放（playback）过程中扮演重要角色，通过快速定位与查询区域相交的绘制操作来减少不必要的渲染计算。

在 Skia 的层次结构中，`SkRTree` 属于核心模块（core）的空间索引子系统，与 `SkBBoxHierarchy` 接口配合工作，为 `SkPicture` 等高层组件提供空间查询能力。

## 主要类与结构体

### SkRTree 类

**继承关系：**
- 继承自 `SkBBoxHierarchy`（抽象基类）

**关键成员变量：**

| 成员变量 | 类型 | 说明 |
|---------|------|------|
| `fCount` | `int` | 数据元素的数量（而非树中的总节点数） |
| `fRoot` | `Branch` | 根分支，包含指向根节点的指针和根边界 |
| `fNodes` | `std::vector<Node>` | 存储所有树节点的连续数组 |
| `kMinChildren` | `static const int` | 每个节点的最小子节点数（值为 6） |
| `kMaxChildren` | `static const int` | 每个节点的最大子节点数（值为 11） |

### 内部结构体

**Node 结构体：**

| 成员变量 | 类型 | 说明 |
|---------|------|------|
| `fNumChildren` | `uint16_t` | 当前节点的子节点数量 |
| `fLevel` | `uint16_t` | 节点在树中的层级（叶节点为 0） |
| `fChildren` | `Branch[kMaxChildren]` | 子分支数组 |

**Branch 结构体：**

| 成员变量 | 类型 | 说明 |
|---------|------|------|
| `fSubtree` / `fOpIndex` | `union` | 内部节点存储子树指针，叶节点存储操作索引 |
| `fBounds` | `SkRect` | 该分支的边界矩形 |

## 公共 API 函数

### 构造与插入

```cpp
SkRTree();
```
默认构造函数，初始化一个空的 R 树。

```cpp
void insert(const SkRect[], int N) override;
```
批量插入 N 个矩形。这是唯一的插入方法，只支持一次性批量加载。调用此方法前树必须为空（`fCount` 必须为 0）。空矩形会被自动过滤掉。

### 查询操作

```cpp
void search(const SkRect& query, std::vector<int>* results) const override;
```
搜索与查询矩形相交的所有操作索引，将结果存储在 `results` 向量中。使用递归遍历树节点，只访问与查询区域相交的分支。

### 信息获取

```cpp
size_t bytesUsed() const override;
```
返回 R 树占用的内存字节数，包括对象本身和节点存储空间。

```cpp
int getDepth() const;
```
返回树的深度。空树返回 0。

```cpp
int getCount() const;
```
返回插入的元素数量（不包括被过滤的空矩形）。

## 内部实现细节

### 批量加载算法（STR）

`bulkLoad()` 方法实现了 STR 算法的核心逻辑：

1. **递归构建**：接收一个分支数组，如果只有一个分支则直接返回作为根
2. **节点分配**：根据 `kMinChildren` 和 `kMaxChildren` 计算需要的节点数
3. **余数处理**：当分支数不能被 `kMaxChildren` 整除时，智能分配余数到其他节点
4. **层级构建**：在当前层级创建节点，然后递归调用自身构建上一层

### 节点计数优化

`CountNodes()` 静态方法在构建前预先计算需要的节点数量，使得 `fNodes` 可以预留足够空间，避免动态扩容导致的性能损失和指针失效问题。

### 空间查询优化

`search()` 方法采用递归深度优先遍历：
- 首先检查根节点的边界矩形与查询矩形是否相交
- 对于内部节点，递归检查每个子分支
- 对于叶节点（`fLevel == 0`），直接将操作索引添加到结果中

### 内存管理

所有节点存储在连续的 `std::vector<Node>` 中，提供良好的缓存局部性。通过预留空间（`reserve()`）避免插入过程中的内存重新分配。

## 设计模式与设计决策

### 批量加载策略

选择只支持批量加载而非增量插入的设计决策基于以下考虑：
- **性能优化**：批量加载可以构建更平衡的树结构
- **使用场景**：Skia 的典型使用场景是先记录所有绘制操作，然后构建索引
- **简化实现**：避免动态插入和删除带来的树重平衡复杂性

### STR 算法选择

采用 STR（Sort-Tile-Recursive）批量加载算法的原因：
- **高效性**：O(n log n) 的构建时间复杂度
- **良好的空间利用率**：节点填充率较高
- **适合静态数据**：Skia 的绘制记录是静态的

代码注释中提到未来可能尝试其他算法（如 Hilbert pack 变体），说明设计者关注算法优化的可能性。

### 经验参数调优

`kMinChildren = 6` 和 `kMaxChildren = 11` 这两个值是通过实验确定的，注释中明确说明"这些值经过实验确定在大多数情况下产生合理的性能"。

### 内存布局优化

使用连续的 `std::vector<Node>` 存储而非指针链表：
- **缓存友好**：连续内存提高缓存命中率
- **减少碎片**：避免大量小对象分配
- **调试断言**：`SkASSERT(fNodes.data() == p)` 确保预留空间足够，没有发生重新分配

## 性能考量

### 构建性能

- **预分配优化**：通过 `CountNodes()` 精确预留所需的节点空间，避免向量动态扩容
- **批量处理**：一次性处理所有数据，减少树调整开销
- **空矩形过滤**：在插入阶段过滤空矩形，减少无效数据

### 查询性能

- **剪枝优化**：通过边界矩形相交测试快速排除不相关的子树
- **平衡树结构**：STR 算法保证树的良好平衡性，查询复杂度接近 O(log n)
- **局部性**：连续的节点存储提高遍历时的缓存性能

### 内存使用

- **紧凑存储**：使用 `uint16_t` 存储节点子节点数量和层级
- **Union 优化**：Branch 结构使用 union 区分叶节点和内部节点，节省空间
- **容量查询**：`bytesUsed()` 方法基于 `capacity()` 而非 `size()`，反映真实内存占用

### 优化建议

注释中提到 Blink 已经提供合理的 x,y 顺序，因此跳过了 Y 轴排序，这在录制阶段带来了 17% 的性能提升，而回放速度几乎没有差异。这体现了针对实际使用场景的优化思想。
