## 概述

`GrAAConvexTessellator` 是专门用于凸多边形抗锯齿细分的类，通过内缩和外扩策略生成带覆盖值的几何。该细分器将凸多边形分解为内部区域（全覆盖）和边缘过渡带（部分覆盖），生成扇形三角形网格，每个顶点携带覆盖属性。适用于凸路径的高质量抗锯齿渲染，比通用三角化器更高效，利用凸性保证几何正确性。

## 架构位置

- **模块层级**：`src/gpu/ganesh/geometry/` - Ganesh 几何处理
- **适用范围**：凸多边形（已知凸性）
- **使用者**：凸路径渲染器、圆角矩形操作
- **输出**：顶点（位置 + 覆盖） + 索引

## 主要类与结构体

### GrAAConvexTessellator

**核心方法**：
```cpp
int tessellate(const SkMatrix& m, const SkPath& path);
```

**顶点属性**：
- 位置（x, y）
- 覆盖值（coverage）

**细分策略**：
- 内缩曲线：沿内法线内缩 0.5px（coverage = 1.0）
- 原始曲线：原始边界（coverage = 可变）
- 外扩曲线：沿外法线外扩 0.5px（coverage = 0.0）

## 内部实现细节

### 环生成

**三层环**：
1. **内环**：内缩 0.5px，全覆盖（coverage = 1.0）
2. **中环**：原始边界，部分覆盖
3. **外环**：外扩 0.5px，零覆盖（coverage = 0.0）

### 扇形三角化

从中心点向外扇形连接，生成三角形扇：
- 内环三角形：全覆盖
- 边缘四边形：分解为两个三角形，覆盖渐变

### 法线计算

**逐边法线**：
- 计算边方向向量
- 旋转 90° 得外法线
- 归一化后缩放 0.5px

**顶点法线**：
相邻两边法线的平均，处理拐角。

### 退化处理

**锐角处理**：
- 检测内缩后的自相交
- 合并过近顶点
- 调整法线避免折叠

## 设计模式与设计决策

### 凸性假设

假设输入为凸多边形，简化算法，避免复杂自相交检测。

### 固定宽度AA

AA 边缘宽度固定为 0.5px（设备空间），保证一致的视觉效果。

### 扇形细分

从中心点扇形细分，三角形数量最少，GPU 友好。

## 性能考量

### 线性复杂度

O(n) 时间复杂度，n 为边数，比通用三角化器快。

### 内存高效

仅存储三层环的顶点，内存占用小。

### GPU 优化

扇形三角形利用 GPU 顶点缓存，提高渲染效率。
