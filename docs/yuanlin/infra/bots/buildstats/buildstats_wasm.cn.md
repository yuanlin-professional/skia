# buildstats_wasm.py - WASM 构建产物大小与符号统计脚本

> 源文件: [infra/bots/buildstats/buildstats_wasm.py](../../../infra/bots/buildstats/buildstats_wasm.py)

## 概述

此脚本用于生成 Skia WASM（WebAssembly）构建产物的详细大小统计信息。除了测量原始大小和 gzip 压缩后大小外，它还使用 `bloaty`（二进制大小分析工具）生成短符号和完整符号两种维度的符号级大小报告。所有统计结果以 Skia Perf 格式输出为 JSON 文件，用于跟踪 WASM 二进制体积的变化趋势。

## 架构位置

该脚本位于 Skia CI 构建统计子系统（`infra/bots/buildstats/`），专门负责 WASM 构建产物的分析。它是构建统计脚本家族中功能介于 `buildstats_web.py`（仅大小）和 `buildstats_cpp.py`（大小 + 段分析）之间的一员，增加了 bloaty 符号分析功能。

## 主要类与结构体

本文件无类定义。`main()` 函数为唯一逻辑入口。

## 公共 API 函数

| 函数 | 参数 | 返回值 | 描述 |
|------|------|--------|------|
| `main()` | 通过 `sys.argv` 接收：input_file, out_dir, keystr, propstr, bloaty_path, total_size_bytes_key, magic_seperator | 无 | 执行符号分析和大小统计 |

## 内部实现细节

1. **参数解析**：接收 7 个位置参数，比 `buildstats_web.py` 多了 `bloaty_path` 参数。
2. **Bloaty 符号分析**（两次调用）：
   - **短符号报告**：`bloaty <file> -d shortsymbols -n 0`，模板实例化被合并，类型被省略
   - **完整符号报告**：`bloaty <file> -d fullsymbols -n 0`，显示完整的模板参数
3. **元数据构建**：解析 key 和 prop 字符串对。
4. **大小测量**：
   - 原始文件大小
   - 复制后 gzip 压缩大小（保护 Swarming 硬链接）
5. **JSON 输出**：将结果写入 `'default'` 配置切片。
6. **日志分段**：使用 magic_seperator 分隔不同的输出段。

## 依赖关系

- **Python 标准库**：`json`、`os`、`subprocess`、`sys`
- **系统工具**：`cp`、`gzip`
- **外部工具**：`bloaty`（Google 的二进制大小分析器，路径通过参数传入）
- **上下游**：CI 构建流水线提供输入，Skia Perf 消费输出

## 设计模式与设计决策

- **双维度符号分析**：分别使用 `shortsymbols` 和 `fullsymbols` 两种模式，前者便于概览（模板被合并），后者便于精确定位。
- **`-n 0` 参数**：告诉 bloaty 输出所有符号而不截断，确保完整的分析数据。
- **信息输出与数据输出分离**：bloaty 的详细报告仅输出到 stdout（供日志查看），而 JSON 数据文件只包含汇总大小。
- **注意事项提示**：脚本输出提示"如果看到大量 func[19] 等信息，请查看 debug 构建"，帮助开发者理解未符号化的条目。

## 性能考量

- bloaty 分析是主要计算开销，对大型 WASM 二进制文件可能耗时数秒。
- 两次 bloaty 调用增加了处理时间但提供了不同粒度的信息。
- gzip 压缩是次要开销。

## 相关文件
