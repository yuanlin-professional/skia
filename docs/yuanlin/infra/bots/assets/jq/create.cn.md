# jq/create.py - jq 命令行工具资产创建脚本

> 源文件: [infra/bots/assets/jq/create.py](../../../../infra/bots/assets/jq/create.py)

## 概述

此脚本用于下载并打包 `jq` 命令行 JSON 处理工具作为 Skia CI 基础设施的 CIPD 资产。它从 GitHub Releases 下载指定版本（v1.7.1）的 jq Linux amd64 二进制文件，验证其 SHA256 哈希值以确保完整性，然后设置执行权限。jq 是 Skia 基础设施中用于解析和操作 JSON 数据的关键工具。

## 架构位置

该脚本属于 Skia CI/CD 资产管理系统（`infra/bots/assets/`），负责为构建机器人提供预打包的工具。jq 工具在 Skia 的自动化流水线中被用于处理配置文件、解析测试结果或操作构建元数据等 JSON 相关任务。

## 主要类与结构体

本文件为简单的脚本工具，不包含类定义。包含以下常量和函数：

- **`URL`**：jq v1.7.1 Linux amd64 版本的 GitHub 下载地址
- **`SHA256`**：预期的 SHA256 哈希值，用于验证下载完整性
- **`BINARY`**：从 URL 中提取的文件名
- **`create_asset(target_dir)`**：核心资产创建函数
- **`main()`**：入口函数

## 公共 API 函数

| 函数 | 参数 | 返回值 | 描述 |
|------|------|--------|------|
| `create_asset(target_dir)` | `target_dir`: 输出目录 | 无 | 下载、验证并安装 jq 二进制文件 |
| `main()` | 无 | 无 | 解析 `--target_dir` 参数并调用创建函数 |

## 内部实现细节

1. **下载**：使用 `wget --quiet` 将 jq 二进制文件下载到目标目录。
2. **哈希验证**：通过 `sha256sum` 命令计算下载文件的哈希值，并与预设的 `SHA256` 常量比较。如果不匹配则抛出异常。
3. **权限设置**：使用 `chmod ugo+x` 为所有用户添加执行权限。

## 依赖关系

- **Python 标准库**：`argparse`、`os`、`subprocess`
- **系统工具**：`wget`（下载）、`sha256sum`（哈希验证）、`chmod`（权限设置）
- **外部资源**：GitHub Releases（`jqlang/jq` 仓库）

## 设计模式与设计决策

- **版本锁定与哈希验证**：通过固定 URL 和 SHA256 哈希确保下载内容的一致性和安全性，防止供应链攻击。
- **简单直接**：脚本采用最小化设计，仅执行下载、验证、设权限三个步骤。
- **使用系统命令**：选择调用 `wget` 和 `sha256sum` 等系统命令而非 Python 库，保持了与其他资产创建脚本的一致性。

## 性能考量

- 脚本的主要耗时在网络下载阶段，受网络带宽影响。
- SHA256 哈希计算对单个二进制文件而言非常快速。
- 仅在需要创建或更新 CIPD 资产时运行。
