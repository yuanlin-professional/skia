# Registry - 自注册链表模板

> 源文件: `tools/Registry.h`

## 概述

`Registry<T>` 是一个模板类,实现了自注册的侵入式链表模式。实例在构造时自动将自己链入全局链表,提供静态遍历接口。用于自动注册图像编解码器、测试用例等服务。

## 架构位置

属于 Skia 工具层的基础组件,被测试框架和编解码器注册系统使用。

## 主要类与结构体

- **`Registry<T>`**: 模板注册表类(SkNoncopyable)
  - `Head()`: 返回链表头
  - `next()`: 返回下一个节点
  - `get()`: 返回存储的值
  - `Range`: 内嵌结构体,支持 range-for 遍历

## 内部实现细节

- 使用静态全局指针 `gHead` 作为链表头
- 构造时将自身插入链表头(LIFO 顺序)
- 支持条件注册(condition 参数)
- Range 迭代器支持 `for (const T& t : Registry<T>::Range()) { ... }` 语法

## 依赖关系

- `include/private/base/SkNoncopyable.h`

## 设计模式与设计决策

- **自注册模式**: 利用全局构造函数在 main() 之前自动注册
- **侵入式链表**: 无需额外内存分配,注册开销极小
- **模板实例化**: 每种 T 类型有独立的链表

## 性能考量

注册为 O(1) 头插入。遍历为 O(n)。无动态内存分配。
