# go.py - Golang 工具封装模块

> 源文件: [tools/infra/go.py](../../tools/infra/go.py)

## 概述

此模块提供了一组 Python 封装函数，用于在 Skia 基础设施脚本中执行 Go 语言相关操作。它包括 Go 环境检查、包获取（`go get`）、模块下载（`go mod download`）和包安装（`go install`）等功能。还提供了专门用于更新 Skia 基础设施代码库（`go.skia.org/infra`）的便捷函数。这是 Skia CI 系统中管理 Go 工具链的核心模块。

## 架构位置

该模块属于 Skia 工具基础设施包（`tools/infra/`），是管理 Go 语言工具的封装层。Skia 的许多基础设施工具（如 bot 管理器、性能跟踪系统等）使用 Go 编写，此模块为 Python 脚本提供了与 Go 工具链交互的标准接口。

## 主要类与结构体

无类定义。关键常量和函数：

- **`INFRA_GO`**：Skia 基础设施 Go 包路径（`'go.skia.org/infra'`）
- **`WHICH`**：平台对应的查找命令（Windows: `'where'`，其他: `'which'`）

## 公共 API 函数

| 函数 | 参数 | 返回值 | 描述 |
|------|------|--------|------|
| `check()` | 无 | 无（失败时退出） | 验证 Go 安装是否正确 |
| `get(pkg)` | `pkg`: Go 包路径 | 无 | 执行 `go get -u` 获取/更新包 |
| `update_infra()` | 无 | 无 | 更新 Skia 基础设施 Go 代码 |
| `mod_download(*pkgs)` | `*pkgs`: 包路径列表 | 无 | 执行 `go mod download` |
| `install(pkg)` | `pkg`: Go 包路径 | 无 | 执行 `go install` |

## 内部实现细节

**`check()` 函数的三步验证**：
1. 使用 `which`/`where` 命令检查 `go` 是否在 PATH 中
2. 检查 `GOPATH` 环境变量是否设置
3. 检查 `$GOPATH/bin` 是否在 `PATH` 中

**内部辅助函数**：
- `_fail(msg)`：向 stderr 输出错误信息并以退出码 1 退出

**各函数行为**：
- `get()` 和 `mod_download()` 和 `install()` 都会先调用 `check()` 验证环境
- `update_infra()` 是 `get(INFRA_GO + '/...')` 的便捷封装

## 依赖关系

- **Python 标准库**：`os`、`subprocess`、`sys`
- **系统工具**：`go`（Go 工具链）、`which`/`where`
- **环境要求**：正确安装的 Go 环境，设置好 `GOPATH` 和 `PATH`

## 设计模式与设计决策

- **前置条件检查**：每个操作函数在执行前都调用 `check()`，确保 Go 环境正确。这是防御性编程的好实践，提供清晰的错误信息而非让 Go 命令本身产生晦涩的错误。
- **快速失败**：`_fail()` 直接调用 `sys.exit(1)` 而非抛出异常，因为 Go 环境不正确通常无法恢复。
- **便捷封装**：`update_infra()` 封装了常用的 Skia 基础设施更新操作，避免其他脚本硬编码 Go 包路径。
- **Python 2 风格**：`print >> sys.stderr` 语法表明这是 Python 2 时代编写的代码。

## 性能考量

- `check()` 每次调用执行一个子进程和两次环境变量查找，开销很小。
- `go get` 和 `go mod download` 涉及网络操作，耗时取决于网络状况。
- 多次调用时 `check()` 会重复执行验证，但开销可忽略。
