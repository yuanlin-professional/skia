# svg_downloader.py - SVG 文件批量下载工具

> 源文件: [tools/svg/svg_downloader.py](../../tools/svg/svg_downloader.py)

## 概述

此脚本用于从 URL 列表文件中批量下载 SVG 文件（及其他图片资源）到指定目录。它支持为下载文件添加自定义前缀，并可选择保留 URL 中公共前缀之后的目录层级结构。该工具主要用于收集 Skia SVG 渲染测试所需的测试素材。

## 架构位置

该脚本属于 Skia 工具层的 SVG 子系统（`tools/svg/`），为 SVG 渲染测试提供测试数据准备能力。它与同目录下的 `svgs.txt` 文件配合使用，后者包含待下载的 SVG 文件 URL 列表。

## 主要类与结构体

无类定义。关键函数和常量：

- **`PARENT_DIR`**：脚本所在目录的绝对路径
- **`download_files(input_file, output_dir, prefix, keep_common_prefix)`**：核心下载函数

## 公共 API 函数

| 函数 | 参数 | 返回值 | 描述 |
|------|------|--------|------|
| `download_files(input_file, output_dir, prefix, keep_common_prefix)` | input_file: URL 列表文件; output_dir: 输出目录; prefix: 文件名前缀; keep_common_prefix: 是否保留目录结构 | 无 | 批量下载文件 |

## 内部实现细节

1. **URL 列表读取**：从输入文件中逐行读取 URL。
2. **目录结构处理**：
   - 如果 `keep_common_prefix` 为 True：计算所有 URL 的公共前缀，保留前缀之后的路径作为目录结构
   - 如果为 False：所有文件下载到同一个平面目录
3. **文件名生成**：`prefix + os.path.basename(file_url)` 构成最终文件名。
4. **目录创建**：使用 `os.makedirs` 按需创建不存在的目录。
5. **下载**：使用 `urllib.urlretrieve` 执行实际下载。

**命令行选项**：
- `-i/--input_file`：URL 列表文件（默认同目录下的 `svgs.txt`）
- `-o/--output_dir`：输出目录（必需）
- `-p/--prefix`：文件名前缀（默认空）
- `-k/--keep_common_prefix`：保留目录结构（默认 False）

## 依赖关系

- **Python 标准库**：`optparse`、`os`、`urllib`
- **输入文件**：包含 URL 列表的文本文件（默认 `tools/svg/svgs.txt`）

## 设计模式与设计决策

- **配置文件驱动**：URL 列表存储在独立的文本文件中，而非硬编码在脚本中，便于添加或修改测试 URL。
- **灵活的目录保留**：`keep_common_prefix` 选项允许保留原始 URL 的目录层级，对于按目录组织的 SVG 测试集很有用。
- **文件名前缀**：支持为下载文件添加前缀，可用于区分不同来源或批次的文件。
- **使用 optparse**：使用较老的 `optparse` 而非 `argparse`，说明脚本编写年代较早（2016 年）。

## 性能考量

- 文件按顺序逐个下载（无并发），对于大量文件可能较慢。
- 使用 `urllib.urlretrieve` 进行简单下载，不支持重试或断点续传。
- 脚本作为一次性数据准备工具使用，性能要求不高。
