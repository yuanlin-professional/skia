# generate_page_set.py - Telemetry 页面集生成器

> 源文件: [tools/skp/generate_page_set.py](../../tools/skp/generate_page_set.py)

## 概述

此脚本是一个交互式工具，用于生成 Chromium Telemetry 框架使用的页面集（page set）Python 文件。页面集定义了需要访问和录制 SKP 的网页 URL、用户代理类型和相关元数据。生成的页面集文件可以被 `webpages_playback.py` 脚本使用，自动访问网页并录制 Skia Picture（SKP）文件，用于渲染性能测试和回归检测。

## 架构位置

该脚本属于 Skia 工具层的 SKP 管理子系统（`tools/skp/`），是 SKP 创建流水线的第一步。它生成页面集定义文件，这些文件随后被 `webpages_playback.py` 使用来实际录制 SKP。生成的文件存放在 `tools/skp/page_sets/` 目录下。

## 主要类与结构体

无类定义。关键常量和函数：

- **`PAGE_SET_TEMPLATE`**：Jinja2 模板文件名（`'page_set_template'`）
- **`PAGE_SET_DIR`**：输出目录（`'page_sets'`）
- **`main()`**：交互式主循环

## 公共 API 函数

| 函数 | 参数 | 返回值 | 描述 |
|------|------|--------|------|
| `main()` | 无（通过 stdin 交互输入） | 无 | 执行交互式页面集创建流程 |

## 内部实现细节

1. **交互循环**：使用 `while True` 循环持续创建页面集直到用户选择停止。
2. **用户输入**：通过 `raw_input()` 收集四项信息：
   - **user_agent**：用户代理类型（mobile/desktop/tablet）
   - **url_name**：URL 的简短名称（如 `google`）
   - **url**：完整的 URL 地址
   - **comment**：添加此 URL 的原因
3. **模板渲染**：使用 Jinja2 模板引擎读取 `page_set_template` 文件并用收集的数据渲染。
4. **文件命名**：输出文件名格式为 `skia_<url_name>_<user_agent>.py`。
5. **结果汇总**：循环结束后列出所有已创建的页面集文件。

## 依赖关系

- **Python 标准库**：`os`
- **Python 第三方库**：`jinja2`（模板引擎）
- **模板文件**：`tools/skp/page_set_template`（Jinja2 模板）
- **Python 版本**：使用 `raw_input()`，仅支持 Python 2

## 设计模式与设计决策

- **模板驱动生成**：使用 Jinja2 模板而非字符串拼接，将页面集的结构与数据分离，模板修改不需要改动脚本。
- **交互式设计**：通过逐步提问收集信息，降低了使用门槛，确保收集到所有必要数据。
- **批量创建**：支持在一次会话中创建多个页面集，减少重复启动脚本的开销。
- **命名约定**：`skia_<name>_<agent>.py` 的命名约定与已有页面集保持一致。
- **注意**：使用 `raw_input()` 表明此脚本仅支持 Python 2，可能需要更新以支持 Python 3。

## 性能考量

- 作为交互式工具，性能不是关注点。
- Jinja2 模板渲染对单个文件来说几乎瞬间完成。

- `tools/skp/page_sets/skia_*_*.py`：已有的页面集定义文件
- `tools/skp/webpages_playback.py`：使用页面集录制 SKP 的工具
