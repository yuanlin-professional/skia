# merge_static_libs.py - 静态库合并工具

> 源文件: [tools/merge_static_libs.py](../../tools/merge_static_libs.py)

## 概述

此脚本用于将多个静态库（`.a` 文件）合并为一个单一的静态库。它支持 POSIX 系统（Linux、macOS）和 Windows 平台，使用各平台原生的归档工具（`ar` 或 `lib`）完成合并操作。在 POSIX 系统上，它能正确处理常规静态库和 thin archive 两种格式。这是构建系统中常用的工具，用于将多个库模块打包为一个发布产物。

## 架构位置

该脚本属于 Skia 工具层（`tools/`），是构建系统的辅助工具。它可能在 Skia 的构建流程中被调用，用于将 Skia 各子模块的静态库合并为单一的 `libskia.a`，简化下游项目的链接过程。

## 主要类与结构体

无类定义。关键函数：

- **`MergeLibs(in_libs, out_lib)`**：核心合并逻辑
- **`Main()`**：入口函数
- **`_Usage()`**：使用帮助

## 公共 API 函数

| 函数 | 参数 | 返回值 | 描述 |
|------|------|--------|------|
| `MergeLibs(in_libs, out_lib)` | `in_libs`: 输入库路径列表; `out_lib`: 输出库路径 | 无 | 合并多个静态库为一个 |
| `Main()` | 无 | 无 | 解析命令行参数并调用合并 |

## 内部实现细节

**POSIX 实现**（`os.name == 'posix'`）：
1. 创建临时目录作为工作区
2. 对每个输入库：
   - `ar -t`：列出库中的目标文件（.o）
   - `ar -x`：尝试提取目标文件
   - 如果提取成功（常规 archive），记录提取文件的绝对路径
   - 如果失败且错误信息包含 `'thin archive'`，使用 `-t` 输出的路径（thin archive 中是文件路径而非嵌入对象）
3. `ar -crs`：将所有收集到的目标文件打包为输出库
4. 清理临时目录

**Windows 实现**（`os.name == 'nt'`）：
- 直接调用 MSVC 的 `lib /OUT:` 命令合并

## 依赖关系

- **Python 标准库**：`os`、`shutil`、`subprocess`、`sys`、`tempfile`
- **环境变量**：`AR`（可选，指定 ar 工具路径，默认 `'ar'`）
- **系统工具**：`ar`（POSIX）或 `lib`（Windows）

## 设计模式与设计决策

- **Thin Archive 处理**：正确区分常规 archive 和 thin archive 两种格式。Thin archive 不包含实际的 .o 文件，只存储路径引用，因此不能用 `ar -x` 提取。
- **临时目录工作区**：在临时目录中提取和操作对象文件，避免文件名冲突和目录污染。
- **环境变量覆盖**：通过 `AR` 环境变量允许指定自定义的 ar 工具路径，支持交叉编译场景。
- **跨平台设计**：通过 `os.name` 分支支持 POSIX 和 Windows 两大平台族。
- **错误处理**：对 `ar` 操作的返回值进行检查，失败时抛出异常。

## 性能考量

- 对大量静态库的合并可能涉及大量磁盘 I/O（提取和重新打包 .o 文件）。
- 使用临时目录避免了内存中操作大文件的问题。
- thin archive 处理更高效，因为不需要实际提取 .o 文件。
- `ar -crs` 的 `s` 标志创建索引，加速后续链接操作。
