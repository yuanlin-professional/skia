# Skia CPU 后端架构

本文档概述了 Skia CPU 后端的主要组成部分。CPU 后端负责在 CPU 上渲染图形，无需使用 GPU。

## 背景知识与定义

*   **位图 (Bitmap)**：由像素网格组成的图像表示，其中每个像素都有一种颜色。这是 Skia CPU 后端绑定的数字画布。常见的图像文件格式如 PNG 和 JPEG 都存储位图数据。
*   **混合模式 (Blend Mode)**：定义源颜色（正在绘制的颜色）如何与目标颜色（画布上已有的颜色）进行数学组合的规则。
*   **光栅化器 (Blitter)**：将渲染管线中的最终像素颜色写入目标位图的组件。它是将着色像素实际放置到画布上的"执行者"。
*   **裁剪 (Clip)**：限制绘制范围的区域。在裁剪区域外绘制的任何内容都不可见。
*   **覆盖遮罩 (Coverage Mask)**：一个值数组（通常为 8 位 alpha 值），表示几何形状覆盖每个像素的程度。值 255 表示完全覆盖，0 表示未覆盖，中间值表示用于抗锯齿的部分覆盖。
*   **发丝线 (Hairline)**：最细的可视线条，通常为一个设备像素宽。画笔 (Paint) 中的描边宽度为 0 被解释为发丝线。发丝线不会随 CTM 缩放。
*   **矩阵 (Matrix)**：用于对绘制内容进行变换（平移、缩放、旋转、倾斜）的数学工具。
*   **画笔 (Paint)**：保存绘制样式信息的对象，如颜色、描边宽度和效果。
*   **路径 (Path)**：描述形状的一系列线条和曲线。Skia 使用二次、三次和圆锥贝塞尔曲线 (Bézier curves) 来创建平滑、可缩放的曲线。二次曲线有一个控制点，三次曲线有两个控制点以实现更复杂的形状。圆锥曲线使用一个控制点和一个权重来表示完美的圆等形状。
*   **光栅化 (Rasterization)**：将矢量图形（基于数学的形状）转换为像素网格（光栅图像）以供显示。
*   **扫描线 (Scanline)**：单行水平像素。复杂形状通常逐扫描线渲染。
*   **着色器 (Shader)**：计算像素颜色的程序。着色器可以生成纯色、渐变、纹理或程序化图案。
*   **分块 (Tiling)**：一种优化策略，将大型绘制操作分解为一系列较小的、分块大小的操作，以保持内存使用（如中间缓冲区）在有限范围内。
*   **环绕数 (Winding Number)**：一种算法，用于确定点是否在复杂或自相交路径内部。想象从该点沿任意固定方向向无穷远画一条射线。环绕数是路径沿一个方向（如顺时针）穿越射线的次数减去沿另一个方向（逆时针）穿越的次数。对于非零环绕规则，任何环绕数非零的点都被视为路径的"内部"。这能正确处理带孔和重叠部分的形状。

## 数据流程详解

以下各节描述了单个绘制命令（如 `canvas.drawPath(...)`）在 CPU 后端中的完整流程，从公共 API 调用到修改原始像素内存。

参见 [CPU.dot](./CPU.dot) 获取补充示意图。

### 1. API 层

流程从面向用户的绘制 API 开始。

- **`SkSurface`**：表示绘制目标。通过 **`SkSurfaces::Raster(...)`** 创建的 CPU 支持的 surface 会通过 `SkPixelRef` 分配并持有像素内存。`SkSurface` 使用此内存管理一个 `SkBitmap`，并提供用于绘制的 `SkCanvas`。
- **`SkImage`**：表示像素数据的不可变快照。它可以从 `SkSurface`（通过 `surface->makeImageSnapshot()`）、`SkBitmap` 或编码数据（如 PNG 文件）创建。在 CPU 后端上，这由 **`SkImage_Raster`** 实现，它持有一个 `SkBitmap`，该 `SkBitmap` 通过 `SkPixelRef` 指向底层像素。因为它是不可变的，所以可以安全地跨线程缓存和共享。可以通过 `canvas->drawImage(...)` 将其绘制到画布上。
- **`SkCanvas`**：主要的绘制接口，提供 `drawPath`、`drawImage` 等方法。用户向画布提供几何图形（如 `SkPath`）、图像（`SkImage`）、样式（`SkPaint`）和字体信息（`SkFont`）。`SkCanvas` 持有当前变换矩阵和裁剪区域。当进行绘制调用时，`SkCanvas` 将所有信息转发给内部的 `SkDevice` 进行渲染。
- **`SkPath`**：表示矢量几何图形（线条、曲线）的高级对象。它是一个轻量级的写时复制对象，持有指向 `SkPathRef` 的共享指针（`sk_sp`）。
- **`SkPathRef`**：实际存储原始路径数据的引用计数对象：点数组、动词（移动、直线、二次曲线、圆锥曲线、三次曲线、闭合）和圆锥权重。`SkPathRef` 对象始终在**堆上分配**并由 `sk_sp` 管理。在内部，`SkPathRef` 使用 `skia_private::STArray` 存储几何数据。这个 `STArray` 是一个小对象优化数组，这意味着对于小路径，几何数据直接存储在 `STArray` 的内部缓冲区中（作为堆分配的 `SkPathRef` 对象的一部分）。对于较大的路径，`STArray` 会在堆上动态分配额外的内存来存储几何数据。

#### 文本渲染

绘制文本是一个专门但常见的用例。该过程涉及三个关键类：
- **`SkFontMgr`**：顶层对象，用于发现和管理系统上安装的字体。它通过匹配字体族名称（如 "Roboto"）或直接从文件或流加载字体数据来创建 `SkTypeface`。它还处理"回退"功能，允许用户从字体集合和排序启发式算法中找到绘制字形的数据（例如，使用此字体绘制拉丁字符，使用另一个字体绘制表情符号）。
- **`SkTypeface`**：表示单个字体面原始数据的不可变对象（如 "Roboto Bold"）。它通常包含每个字形的矢量路径，但也支持位图字形。矢量字形会被光栅化并存储在字形缓存（`SkStrikeCache`）中。
- **`SkFont`**：一个轻量级对象，持有对 `SkTypeface` 的引用以及文本大小、缩放和倾斜等样式属性。

当用户调用 `canvas.drawSimpleText(...)` 等方法时，他们向 `SkCanvas` 提供文本和 `SkFont`。后端随后使用 `SkFont` 从 `SkTypeface` 查找各个字形。每个字形随后被视为标准 `SkPath` 进行渲染。

**关于文本排版的说明：** Skia 核心处理单个字形的渲染，但不执行复杂的文本排版（如换行、对齐或双向文本）。对于富文本排版，Skia 提供了 **`SkParagraph`** 模块，这是一个构建在 Skia 核心组件之上的高级库。

#### 画笔配置

**`SkPaint`** 对象持有一组可选组件，用于控制样式和像素处理管线。
- **`SkShader`**：为几何图形生成源颜色。如果没有着色器，则使用画笔的颜色。着色器可以生成渐变、位图图案或程序化颜色。
- **`SkColorFilter`**：修改由着色器或画笔生成的源颜色。常见用途包括着色或应用颜色矩阵进行颜色校正。
- **`SkMaskFilter`**：作用于形状的覆盖遮罩而非颜色。其最常见的用途是模糊形状边缘，例如通过 `SkMaskFilter::MakeBlur` 进行高斯模糊。
- **`SkPathEffect`**：在绘制前修改形状的几何图形。例如，`SkPathEffect::MakeDash` 创建将实线变为虚线的效果。这发生在光栅化之前。
- **`SkBlender`**：控制源颜色（来自着色器）如何与目标颜色（画布上已有的颜色）混合。它是传统 `SkBlendMode` 枚举的更强大、可编程的版本。混合逻辑作为 `SkRasterPipeline` 中的一个阶段执行。
- **`SkImageFilter`**：对绘制操作的输出应用复杂的多遍效果。与其他效果不同，图像滤镜可以将整个绘制结果作为纹理进行操作。这通常需要分配一个临时的离屏图层。例如，投影阴影滤镜可能会将形状绘制到图层中，对其进行模糊、偏移，然后在模糊的阴影上方再次绘制原始形状。

### 2. 设备层：`SkBitmapDevice`

对于 CPU 后端，`SkCanvas` 将绘制调用转发给 **`SkBitmapDevice`**。此对象是所有基于 CPU 绘制的具体目标。它管理目标 **`SkBitmap`** 并通过创建和分派 `SkDraw` 对象来启动渲染。

**关于裁剪的说明：** 设备管理一个与画布的 `save()`/`restore()` 调用对应的 `SkRasterClipStack`。对于每次绘制，它将此栈解析为单个 `SkRasterClip` 并传递给 `SkDraw`，后者对裁剪是无状态的。

**关于分块的说明：** 对于大型或复杂的绘制，设备可能使用分块技术。它将操作分解为较小的分块大小的块，为每个块调整裁剪区域。这通过为每个分块调用一次绘制过程来限制中间缓冲区所需的内存。

### 3. 编排器：`SkDraw`

`SkBitmapDevice` 为每个图元创建一个 **`SkDraw`** 对象。`SkDraw` 编排单次绘制，捆绑几何图形（`SkPath`）、样式（`SkPaint`）、变换、裁剪和目标 `SkPixmap`。它使用 `SkScan` 将矢量形状转换为光栅操作。

### 4. 光栅化器：`SkScan`

`SkDraw` 将 `SkPath` 传递给 **`SkScan`**——核心光栅化器。`SkScan` 将矢量路径转换为水平扫描线，为每条扫描线计算覆盖遮罩（alpha 值）。它不感知颜色或效果。`SkDraw` 向 `SkScan` 提供一个 `SkBlitter`，`SkScan` 为每条扫描线调用该 blitter 来渲染覆盖数据。

**关于扫描转换器的说明：** `SkPaint` 上的 `isAntiAlias()` 标志选择扫描转换器：
- **锯齿模式 (`SkScan_Path.cpp`)：** 当抗锯齿 (AA) 关闭时，它根据像素中心是否在路径内部生成二值（0% 或 100%）覆盖遮罩，产生硬边缘。对于每段连续覆盖的水平像素，它调用 `blitter->blitH(x, y, width)`。
- **解析抗锯齿 (`SkScan_AAAPath.cpp`)：** 当 AA 开启时，它精确地解析计算与每个像素的几何交集面积，产生分数覆盖值以实现平滑边缘。对于每段部分覆盖的水平像素，它调用 `blitter->blitAntiH(x, y, alphas, runs)`。

### 5. 像素写入器：`SkRasterPipelineBlitter`

**`SkRasterPipelineBlitter`** 是 CPU 后端中的主要 `SkBlitter`。它实现了 `SkBlitter` 接口，但不直接写入像素，而是将来自 `SkScan` 的 `blitH` 或 `blitRect` 等调用转换为其内部 `SkRasterPipeline` 的执行。它使用来自 blitter 调用的正确覆盖信息配置管线，然后运行管线来计算最终像素颜色并写入相应行。

### 6. 核心引擎：`SkRasterPipeline`

**`SkRasterPipeline`** 通过链接单一用途的**阶段 (stages)** 来计算最终像素颜色。例如，一次绘制可能使用加载覆盖（`load_a8`）、设置源颜色（`uniform_color`）、加载目标、混合（`srcover`）和存储结果（`store_8888`）等阶段。这种基于阶段的设计避免了函数的组合爆炸。

管线被组装成一系列预编译的"阶段"，在覆盖区域上循环执行。这些函数使用 **SIMD（如 SSE、NEON）** 一次处理多个像素以获得高性能。管线从 `SkPixmap` 获取内存布局信息以加载和存储像素。

### 7. 像素内存视图：`SkPixmap`、`SkBitmap` 和 `SkPixelRef`

这是像素被修改的最终站点。
- **`SkPixmap`**：一个轻量级对象，包含指向像素内存的指针及其元数据（尺寸、颜色类型）。它为 `SkRasterPipeline` 提供读写的精确内存地址。
- **`SkBitmap`**：由 `SkBitmapDevice` 持有，它将 `SkImageInfo` 与像素存储配对。
- **`SkPixelRef`**：拥有堆分配像素内存的智能指针，最初由 `SkSurface` 创建。

`SkRasterPipeline` 的最终阶段使用来自 `SkPixmap` 的地址将新颜色写入 `SkPixelRef` 拥有的内存中，完成绘制。
