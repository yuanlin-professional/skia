GPU 上的渐变 (Gradients on the GPU)
====================

在非常高的层次上，渐变可以被认为由三个部分组成：

1. 一个一维的颜色插值器 (Colorizer)，对于 [0.0, 1.0] 范围内的插值参数 "t" 返回一个颜色。它封装了特定颜色停止点的定义，以及如何处理越界输入的环绕、平铺或钳位操作。颜色插值器的命名格式为 `GrXxxxxGradientColorizer`。
2. 一个布局 (Layout)，将二维几何/位置转换为颜色插值器的一维域。线性渐变或径向渐变正是通过此方式区分自身。在设计新的渐变形状时，这是需要实现的组件。布局的命名格式通常为 `GrYyyyyGradientLayout`。
3. 一个顶层效果 (Top-level Effect)，将布局和颜色插值器组合在一起。此处理器还负责实现钳位行为，该行为对布局和着色都是抽象的。

`GrClampedGradientEffect` 处理钳位和衰减平铺模式 (Decal Tile Mode)，而 `GrTiledGradientEffect` 实现重复和镜像平铺模式。`GrClampedGradientEffect` 要求在其颜色插值器子节点之外指定边界颜色，但这些边界颜色可以由渐变颜色停止点定义。这两个顶层效果都将 t 插值参数的计算委托给布局子处理器，然后执行各自的平铺模式操作，最后使用颜色插值器子处理器将平铺后的 t 值（保证在 0 和 1 之间）转换为输出颜色。

片段处理器 (Fragment Processor) 仅支持返回颜色；然而从概念上讲，布局处理器需要生成一个插值参数，而不是颜色。因此布局处理器将其结果编码到颜色中，方式如下：

-  `sk_OutColor.r`：计算得到的 t 插值参数 [0.0, 1.0]，未平铺
-  `sk_OutColor.g`：正值 = 渲染，负值 = 丢弃像素
-  `sk_OutColor.b`：未使用
-  `sk_OutColor.a`：未使用

布局可以通过向 `sk_OutColor.g` 分量输出负值来报告"无效渐变位置"。（目前，双点锥形渐变会这样做。）当这种情况发生时，顶层效果立即返回透明黑色，完全不调用颜色插值器。当渐变位置有效时，顶层效果在显式坐标 (t, 0) 处从颜色插值器采样。y 坐标始终为零，颜色插值器可以忽略它。

有几个手写的颜色插值器用于解析式颜色情况；这些直接在着色器 (Shader) 中求值。生成的纹理贴图也可以用于着色渐变；在这种情况下，会使用 `GrTextureEffect` 作为颜色插值器。

`GrGradientShader` 提供静态工厂函数来创建能够复现特定 `SkGradientShader` 的 `GrFragmentProcessor` 图。

优化标志 (Optimization Flags)
==================

在抽象层面上，渐变着色器与覆盖率作为 Alpha 兼容，并且在某些条件下，当输入不透明时能保持不透明性。为了减少重复代码和样板代码，这些优化决策在顶层效果中实现，而不是在颜色插值器中。假设所有颜色插值器片段处理器都与覆盖率作为 Alpha 兼容，并且在输入颜色不透明时保持不透明性。由于这是顶层效果所假设的，因此它们不需要报告这些优化或检查输入不透明性（这确实意味着如果颜色插值器独立于顶层效果着色器使用，报告的标志可能不是最优的，但由于这种情况不太可能发生，这个约定确实简化了颜色插值器的实现）。

与不需要报告任何优化标志的颜色插值器不同，布局片段处理器应该报告保持不透明性的优化，因为它们可以影响渐变以外方式着色的像素的不透明性。可能拒绝像素（即可能输出负 y 值）的布局不得报告 kPreservesOpaqueInput_OptimizationFlag。永远不拒绝像素的布局应该报告 kPreservesOpaqueInput_OptimizationFlag，因为顶层效果可以优化掉检查布局是否拒绝像素的操作。
