色彩管理的含义
--------------------------------

Skia 使用的所有色彩空间 (color space) 都通过描述如何将颜色从该色彩空间转换
到一个称为 XYZ D50 的通用"连接"色彩空间来定义自身。我们也可以从同样的描述
中推断出如何从 XYZ D50 空间转换回原始色彩空间。XYZ D50 是一个类似 RGB 的
三维色彩空间，但 XYZ 部分并不像 RGB，而是这些通道的线性重混合。Y 最接近
你所认为的亮度，但 X 和 Z 更加抽象。如果你熟悉 YUV 的话，它有点像 YUV。
"D50"部分指的是该空间的白点 (whitepoint)，约为 5000 开尔文。

所有色彩管理绘制分为六个部分，三个步骤将源颜色连接到 XYZ D50 空间，
然后三个对称的步骤从 XYZ D50 连接回目标色彩空间。
其中一些步骤可以相互抵消变为空操作 (no-op)，有时当源空间和目标空间相同时，
整个过程都会变成空操作。以下是这些步骤：

色彩管理步骤
----------------------

1. 如果源颜色是预乘的 (premultiplied)，则取消预乘 -- alpha 不参与
   色彩管理，如果它已被乘入，我们需要将其除出来
2. 使用源色彩空间的传递函数 (transfer function) 将源颜色线性化
3. 通过乘以一个 3x3 矩阵，将未预乘的线性源颜色转换到 XYZ D50 色域 (gamut)
4. 通过乘以一个 3x3 矩阵，将 XYZ D50 颜色转换到目标色域
5. 使用目标色彩空间传递函数的逆函数对该颜色进行编码
6. 如果目标是预乘的，则按 alpha 进行预乘

如果你查看我们的代码，看到这个逻辑最清晰的地方是一个名为
SkColorSpaceXformSteps 的类型。你会看到它有 5 个步骤：我们总是
将最内层的两个操作合并为一个 3x3 矩阵乘法。

优化
-------------

每当我们即将进行绘制时，我们会查看哪些步骤是真正需要的。
任何本质上是空操作的步骤我们都会跳过：

   * 如果源已经是未预乘的，跳过步骤 1
   * 如果源已经是线性编码的，跳过步骤 2
   * 如果单个合并矩阵是单位矩阵 (identity)（即源和目标色彩空间具有相同的色域），跳过步骤 3 和 4
   * 如果目标需要线性编码，跳过步骤 5
   * 如果目标需要未预乘，跳过步骤 6

我们可以从这些基本跳过推导出一些更高级的优化：

  * 如果已经跳过了步骤 3 和 4，只要传递函数相同，我们就可以跳过步骤 2 和 5 -- 将颜色通过一个给定传递函数及其自身的逆函数是空操作
  * 如果已经跳过了步骤 2-5，如果两者都要执行，我们就可以跳过步骤 1 和 6 --- 没必要取消预乘后又重新预乘
  * 不透明颜色可以被视为未预乘或预乘的，取决于哪种方式能让我们跳过更多步骤

这一切汇聚在一起，令人印象深刻的是大多数时候"无事可做"。
如果你在给定的色彩空间中绘制不透明颜色到标记了相同色彩空间的目标，
我们会注意到可以跳过所有六个步骤。有时需要更少的步骤，有时需要更多。
一般来说，如果你需要进行色域转换，你通常应该预期所有中间步骤都会被激活。
步骤 2 和 5 是计算开销最大的。

nullptr SkColorSpace 默认值
-----------------------------

那么 nullptr SkColorSpace 默认值在所有这些中如何运作？我们在上面提到的所有
逻辑之前加上这小段代码：

     if (srcCS == nullptr) { srcCS = sRGB; }
     if (dstCS == nullptr) { dstCS = srcCS; }

（顺序在这里很重要。）要点是，我们假设任何未标记的源是 sRGB。
如果你让你的表面 (surface) 未标记，我们会表现得好像你的目标流畅地
匹配你试图绘制到其中的任何源，这至少会跳过上面列出的步骤 2-5，
维持一种与 Skia 在引入色彩管理之前的旧有工作方式兼容的非管理色彩绘制模式。
这不是很有原则性，但在实践中保留它很方便。
