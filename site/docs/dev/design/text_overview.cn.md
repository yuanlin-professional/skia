*2D 图形*是一个广泛的领域。它由一系列相关但不同的概念和基本图元 (primitive) 组成
- 矩形、线段、路径
- 图像（通常经过缩放、变换、滤镜处理）
- 文本（同样可缩放和可变换）
- ...

原生（桌面、移动端）框架将这些基本图元与通用渲染构造相结合
- 变换和裁剪
- 颜色、渐变、图案
- 混合和色彩空间

有一个基本图元特别突出：**文本**。它可以使用与矩形和路径相同的通用构造来绘制，但它还有一个与实际绘制无关的组成部分——一个出奇复杂且计算开销大的"处理"步骤：**文本整形 (Shaping)**。原生框架认识到了这一点，并为此提供了一组单独的 API：
- CoreGraphics --> CoreText
- Direct2D/X --> DirectWrite
- Skia --> SkShaper

本提案采用这一模式，旨在将此功能暴露给 Web。


## 背景

这种复杂性的部分原因在于语言和字母表本身，部分原因在于我们的字体和字体格式系统的演变方式。本文档不打算详细说明这两方面，但会简要涉及以引出提议的 API 套件。

## 语言和字母表

### "The office 大楼 is in München."

这个简单的句子说明了文本显示的一些丰富性（即复杂性）。一些示例
- office 中的 "ffi"（或者可能只是 "fi"）**可以**用一种称为[连字 (ligature)](https://en.wikipedia.org/wiki/Ligature_(writing)) 的专门形式来绘制。
- "ü" 可以作为单个形状绘制，也可以通过先绘制 "u" 然后在其上方绘制分音符（¨）来分别绘制。
- 第三个词可能无法用与其他词相同的字体表示，因此可能需要使用不同的字体。

这个示例绝不试图完整或详尽，但它暗示了从句子中的"字母"到必须从各种字体中找到或组合的实际"形状"所需的处理过程。

## 字体和样式

这个[网站](https://www.w3schools.com/css/css_font.asp)对在 CSS 中指定"字体"的一些丰富性有很好的概述。虽然不同的原生平台有一些差异，但大多数支持以下从字体描述到包含绘制形状的实际资源/文件/二进制数据的*映射*。

- family-name（字体族名称）: "Helvetica", "Times New Roman", "Lucida Typewriter Sans", ...
- font-style（字体样式）: bold, italic
- font-weight（字体粗细）: 100 .. 900
- font-stretch（字体拉伸）: ultra-condensed .. ultra-expanded
- font-variation-setting（字体变体设置）: 'wght' 850 or 'wdth' 25 or ...

所有这些属性（以及更多）都会参与浏览器对最佳匹配字体资源/文件/二进制数据的搜索。我们将结果"资源"称为**字型 (Typeface)**。注意：如果字体支持 variation-settings，我们的定义还将包括特定设置（例如字体资源 + 变体设置）。

## 字型和字形 (Typefaces and Glyphs)

我们将字型 (Typeface) 定义为给定排版风格（包括任何变体设置）中一组协调的形状集合。通常字型存储在单个文件或二进制数据中（例如 OpenType 文件）。

与单个字型一起，我们将字型中的各个*形状*或图像定义为**字形 (Glyph)**。字形代表字型中最小的独立绘制元素。按照惯例，字形由一个*索引*标识，范围从 0 到特定字型中包含的数量。

确定需要哪些字形、以什么顺序和位置来表示一组字母，这就是**文本整形 (Shaping)** 的核心，而我们提议向 Web 应用暴露的正是这个过程及其结果——字型 + 定位后的字形。

## 总结

我们认为绘制*国际化正确*的文本对大多数 Web 应用至关重要，而且正确实现既复杂又可能计算开销很大。我们提议向应用暴露此处理过程，为它们提供可以高效绘制/动画化的结果。

核心[文本整形 API](/docs/dev/design/text_shaper) 在此详细说明。

关联的 [Canvas2D 扩展](/docs/dev/design/text_c2d)在此详细说明。

注意：一个明确的目标是**不**将文本整形或其结果绑定到 Canvas2D。我们设想了多种场景，其中框架或应用需要对文本进行整形，但不需要 Canvas2D 上下文。
- 使用 WebGL 或 WebGPU 绘制
- 使用 DOM 绘制（但利用换行和度量结果）
- 使用定制的（即 wasm）渲染器绘制

我们还提议了一个更低级的接口，仅用于暴露 [Unicode 属性](/docs/dev/design/uni_characterize)。
